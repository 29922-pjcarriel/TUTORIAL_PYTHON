Manejo de Errores y Excepciones

Estructura de manejo de errores o control de excepciones en PHP y Python.

1 Estructura General

Ambos lenguajes utilizan un mecanismo similar basado en bloques para capturar y manejar errores en tiempo de ejecución. La diferencia clave está en las palabras reservadas.

ConceptoPHPPython
Bloque de intentotry { }try:
Captura de errorescatch (Exception $e) { }except ValueError as e:
Bloque finalfinally { }finally:
Lanzar excepciónthrow new Exception()raise Exception()

2 PHP: try / catch / finally

PHP

"Pruebo el código en TRY, capturo errores con CATCH y siempre ejecuto FINALLY."

try {
    $entrada = readline("Ingresa un número: ");
    if (!is_numeric($entrada)) {  // Verifica si no es número
        throw new Exception("Entrada no es un número");
    }
    $numero = (int)$entrada;
    echo "El doble es: " . ($numero * 2) . "\n";
} catch (Exception $e) {
    echo "¡Eso no es un número! " . $e->getMessage() . "\n";
} finally {
    echo "Gracias por participar\n";
}

$e es el objeto de excepción con métodos para acceder al error, como $e->getMessage().

3 Python: try / except / finally

Python

"Intento el código en TRY, manejo errores en EXCEPT y siempre ejecuto FINALLY."

try:
    numero = int(input("Ingresa un número: "))
    print("El doble es:", numero * 2)
except ValueError as e:
    print("¡Eso no es un número!", e)
finally:
    print("Gracias por participar")

e es el objeto de excepción con información del error. Python usa except en lugar de catch.
