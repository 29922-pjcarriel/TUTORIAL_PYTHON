1 Sesiones

PHP
  • El cliente no puede leer los datos de la sesión.
  • El cliente no puede modificar el contenido de la sesión.
  • El navegador únicamente almacena un identificador de sesión (PHPSESSID).
  • Los datos reales de la sesión residen exclusivamente en el servidor.
Python + Flask
  • El cliente puede leer el contenido de la sesión, ya que se almacena en una cookie.
  • Los datos se encuentran codificados (por ejemplo, Base64), pero no cifrados.
  • El cliente no puede modificar los datos sin invalidar la firma criptográfica.
  • La integridad de la sesión está protegida mediante la secret_key.

2 Almacenamiento de Sesión

PHP
  • El navegador almacena únicamente una cookie con el identificador de sesión (PHPSESSID).
  • El servidor utiliza dicho identificador para localizar el archivo de sesión correspondiente.
  • Los datos reales de la sesión se encuentran almacenados en el servidor, no en el cliente.
// Iniciar sesión
session_start();

// Almacenar datos
$_SESSION['usuario'] = 'admin';
$_SESSION['listaNote'] = $notebooks;
Python + Flask
  • El navegador guarda una cookie que contiene los datos de sesión serializados.
  • La información está firmada criptográficamente mediante la secret_key.
  • El servidor valida la firma, pero no conserva estado entre solicitudes.
# Configurar clave secreta
app.secret_key = 'mi_clave_secreta'

# Almacenar datos
session['usuario'] = 'admin'
session['listaNote'] = notebooks

4 Flujo de Ejecución en Python

Python
  1. El usuario accede a la URL principal de la aplicación (/).
  2. Flask detecta la ruta / y ejecuta la función root(), que es el primer punto de entrada del sistema.
  3. Dentro de la función, el servidor localiza el archivo index.html que se encuentra en la raíz del proyecto.
  4. El archivo index.html se abre y su contenido se lee completamente en memoria.
  5. Ese contenido se renderiza usando Jinja, lo que permite interpretar expresiones como url_for para cargar correctamente estilos e imágenes.
  6. Finalmente, Flask envía la página ya procesada al navegador del usuario, mostrando la pantalla inicial del sistema.
¿Qué es render_template_string?
return render_template_string(content)
  • Tomar un HTML
  • Procesarlo con Jinja (el motor de plantillas)
  • Convertirlo en una página lista para el navegador
Archivo: app.py — Imports y configuración
from flask import Flask, render_template, render_template_string, request, session, url_for
from Conexion import Conexion
from Notebook import Notebook
import os

app = Flask(__name__,
            template_folder=os.path.join(os.getcwd(), 'templates'),
            static_folder=os.path.join(os.getcwd(), 'static'))
app.secret_key = 'clave_secreta_flask'
Función root() — Punto de entrada
@app.route('/')
def root():
    file_path = os.path.join(os.getcwd(), 'index.html')
    with open(file_path, 'r', encoding='utf-8') as f:
        content = f.read()
    return render_template_string(content)

8 Obtención de Datos y Almacenamiento en Sesión

Python
  • Se usa para enviar información al servidor.
  • Modifica o procesa datos.
  • Los datos no viajan en la URL.
  • Se usa en formularios (login).
# app.py — Ruta /index
@app.route('/index')
def index():
    conn = Conexion()
    cursor = conn.getConexion().cursor(dictionary=True)
    cursor.execute("SELECT * FROM notebook")
    rows = cursor.fetchall()

    notebooks = []
    for row in rows:
        n = Notebook(row['marca'], row['codigo'], row['precio'])
        notebooks.append(n.__dict__)

    # Almacenar en sesión
    session['listaNote'] = notebooks
    return render_template('index.html', notebooksData=notebooks)
PHP
  • __DIR__ es una constante mágica de PHP.
  • Representa la ruta absoluta del directorio actual.
  • Se usa para evitar errores de rutas relativas.
// index.php — Obtención de datos
<?php
$sql = "SELECT * FROM notebook";
$resultado = mysqli_query($conn, $sql);

$notebooks = [];
while ($row = mysqli_fetch_assoc($resultado)) {
    $obj = new Notebook(
        $row['marca'],
        $row['codigo'],
        $row['precio']
    );
    $notebooks[] = $obj;
}

// Almacenar en sesión
$_SESSION['listaNote'] = $notebooks;

include __DIR__ . '/../frontend/templates/index.php';
?>

9 Generación Dinámica del Formulario

Python
  • Usa Jinja ({% for %}), para generar HTML dinámico.
  • El HTML solo presenta datos enviados por el backend.
  • Inserta los valores directamente en la plantilla.
  • Usa variables del diccionario.
  • Llama a una ruta del backend.
<!-- templates/index.html (Jinja) -->
<form action="/verNotebook" method="POST">

    {% for item in notebooksData %}
    <option value="{{ loop.index0 }}">
      {{ item.marca }} - ${{ item.precio }}
    </option>
    {% endfor %}
  </select>
  <button type="submit">Ver Detalle</button>
</form>
PHP
  • Usa foreach dentro del HTML.
  • Ejecuta código PHP embebido en la vista.
  • Genera las opciones con echo.
  • Accede a métodos del objeto.
  • Llama directamente a un archivo PHP.
<!-- frontend/templates/index.php -->
<form action="verNotebook.php" method="POST">
  <select name="op">
    <?php
    foreach ($notebooks as $key => $obj) {

10 Control del Flujo y Visualización de Datos

Python
  • Utiliza una ruta (/verNotebook) definida en Flask para controlar el acceso.
  • Flask decide qué función ejecutar según la ruta y el método (GET o POST).
  • Recupera los datos almacenados en la sesión usando session.get.
  • Lee la información enviada desde el formulario mediante request.form o request.args.
  • Selecciona el elemento correspondiente desde la estructura de datos.
  • Envía los datos a la vista utilizando render_template.
# app.py
@app.route('/verNotebook', methods=['GET', 'POST'])
def ver_notebook():
    notebooks = session.get('listaNote', [])

    if request.method == 'POST':
        op = int(request.form['op'])
    else:
        op = int(request.args.get('op', 0))

    objData = notebooks[op]
    return render_template('verNotebook.html', objData=objData)
PHP
  • El flujo se controla mediante un archivo PHP accedido directamente.
  • El archivo se ejecuta cuando el formulario apunta a él.
  • Recupera los datos almacenados en la sesión usando $_SESSION.
  • Lee la información enviada desde el formulario mediante $_POST o $_GET.
  • Selecciona el elemento correspondiente desde el arreglo de datos.
  • Incluye directamente la vista que mostrará la información al usuario.
// verNotebook.php
<?php
session_start();

$notebooks = $_SESSION['listaNote'];

if (isset($_POST['op'])) {
    $op = $_POST['op'];
} else {
    $op = $_GET['op'];
}

$obj = $notebooks[$op];

include __DIR__ . '/../frontend/templates/verNotebook.php';
?>

15 Validación de Login y Manejo de Sesión

Python
Login
  • El formulario envía los datos (usuario y clave) mediante POST a la ruta /validar.
  • Flask recibe los datos usando request.form.
  • Se ejecuta una consulta a la base de datos para verificar si el usuario existe.
  • Si los datos son correctos: El login se considera válido.
  • Si los datos son incorrectos: Se renderiza una vista de error.
Sesión
  • Flask utiliza el objeto session.
  • La sesión se guarda usando una clave secreta (secret_key).
# app.py
@app.route('/validar', methods=['POST'])
def validar():
    usuario = request.form['usuario']
    clave = request.form['clave']

    conn = Conexion()
    cursor = conn.getConexion().cursor(dictionary=True)
    sql = "SELECT * FROM usuarios WHERE user=%s AND pass=%s"
    cursor.execute(sql, (usuario, clave))
    result = cursor.fetchone()

    if result:
        session['usuario'] = usuario
        notebooks = session.get('listaNote', [])
        return render_template(
            'index.html',
            notebooksData=notebooks
        )
    else:
        return render_template('ErrorAutenticacion.html')
PHP
Login
  • El formulario envía los datos mediante POST al archivo validar.php.
  • PHP recibe los datos usando $_POST.
  • Se ejecuta una consulta SQL para verificar usuario y contraseña.
  • Si los datos son correctos: Se redirige al siguiente archivo.
  • Si los datos son incorrectos: Se incluye una vista de error.
Sesión
  • PHP maneja sesiones con session_start().
  • Los datos se guardan en la superglobal $_SESSION.
// validar.php
<?php
session_start();
require_once __DIR__ . '/../Conexion.php';

$usuario = $_POST['usuario'];
$clave = $_POST['clave'];

$conexion = new Conexion();
$conn = $conexion->getConexion();

$sql = "SELECT * FROM usuarios
        WHERE user='$usuario' AND pass='$clave'";
$resultado = mysqli_query($conn, $sql);

if (mysqli_num_rows($resultado) > 0) {
    $_SESSION['usuario'] = $usuario;
    $notebooks = $_SESSION['listaNote'];
    include __DIR__ . '/../frontend/templates/index.php';
} else {
    include __DIR__ . '/../frontend/templates/ErrorAutenticacion.php';
}
?>
