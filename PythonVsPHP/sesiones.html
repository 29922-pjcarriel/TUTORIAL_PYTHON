Sesiones y Login

Comparativa de manejo de sesiones, autenticación y base de datos entre Python (Flask) y PHP.

1 Sesiones

PHP
  • El cliente no puede leer los datos de la sesión.
  • El cliente no puede modificar el contenido de la sesión.
  • El navegador únicamente almacena un identificador de sesión (PHPSESSID).
  • Los datos reales de la sesión residen exclusivamente en el servidor.
Python + Flask
  • El cliente puede leer el contenido de la sesión, ya que se almacena en una cookie.
  • Los datos se encuentran codificados (por ejemplo, Base64), pero no cifrados.
  • El cliente no puede modificar los datos sin invalidar la firma criptográfica.
  • La integridad de la sesión está protegida mediante la secret_key.

2 Almacenamiento de Sesión

PHP
  • El navegador almacena únicamente una cookie con el identificador de sesión (PHPSESSID).
  • El servidor utiliza dicho identificador para localizar el archivo de sesión correspondiente.
  • Los datos reales de la sesión se encuentran almacenados en el servidor, no en el cliente.
Python + Flask
  • El navegador guarda una cookie que contiene los datos de sesión serializados.
  • La información está firmada criptográficamente mediante la secret_key.
  • El servidor valida la firma, pero no conserva estado entre solicitudes.

3 Organización del Proyecto

La estructura del proyecto separa claramente el frontend y el backend en ambas implementaciones: PHP y Python. Esto facilita la organización del código y el mantenimiento del sistema.

PHP

Cada archivo PHP actúa como parte del backend, mezclando lógica y presentación según el archivo.

Python

Flask separa las rutas (backend) de las plantillas HTML (frontend) mediante el motor Jinja.

4 Flujo de Ejecución en Python

Python
  1. El usuario accede a la URL principal de la aplicación (/).
  2. Flask detecta la ruta / y ejecuta la función root(), que es el primer punto de entrada del sistema.
  3. Dentro de la función, el servidor localiza el archivo index.html que se encuentra en la raíz del proyecto.
  4. El archivo index.html se abre y su contenido se lee completamente en memoria.
  5. Ese contenido se renderiza usando Jinja, lo que permite interpretar expresiones como url_for para cargar correctamente estilos e imágenes.
  6. Finalmente, Flask envía la página ya procesada al navegador del usuario, mostrando la pantalla inicial del sistema.
¿Qué es render_template_string?
return render_template_string(content)
  • Tomar un HTML
  • Procesarlo con Jinja (el motor de plantillas)
  • Convertirlo en una página lista para el navegador

5 Head del Index.html

PHP
  • Es un archivo HTML estático (.html).
  • No depende de ningún framework.
  • Los recursos (CSS, imágenes) se cargan con rutas directas.
Python
  • Es una plantilla HTML renderizada por el backend.
  • Depende de un framework (por ejemplo, Flask).
  • Los recursos se cargan de forma dinámica usando el backend.

  • Importa módulos propios del proyecto.

9 Generación Dinámica del Formulario

Python
  • Usa Jinja ({% for %}), para generar HTML dinámico.
  • El HTML solo presenta datos enviados por el backend.
  • Inserta los valores directamente en la plantilla.
  • Usa variables del diccionario.
  • Llama a una ruta del backend.
PHP
  • Usa foreach dentro del HTML.
  • Ejecuta código PHP embebido en la vista.
  • Genera las opciones con echo.
  • Accede a métodos del objeto.
  • Llama directamente a un archivo PHP.

10 Control del Flujo y Visualización de Datos

Python
  • Utiliza una ruta (/verNotebook) definida en Flask para controlar el acceso.
  • Flask decide qué función ejecutar según la ruta y el método (GET o POST).
  • Recupera los datos almacenados en la sesión usando session.get.
  • Lee la información enviada desde el formulario mediante request.form o request.args.
  • Selecciona el elemento correspondiente desde la estructura de datos.
  • Envía los datos a la vista utilizando render_template, renderizando el HTML dinámicamente.
PHP
  • El flujo se controla mediante un archivo PHP accedido directamente.
  • El archivo se ejecuta cuando el formulario apunta a él.
  • Recupera los datos almacenados en la sesión usando $_SESSION.
  • Lee la información enviada desde el formulario mediante $_POST o $_GET.
  • Selecciona el elemento correspondiente desde el arreglo de datos.
  • Incluye directamente la vista que mostrará la información al usuario.

11 Vista de Detalle del Notebook

Python
  • Es una plantilla HTML con Jinja.
  • Los datos llegan desde el backend mediante render_template.
  • Se accede a los valores usando variables dinámicas.
{{ objData.marca }}
{{ objData.codigo }}
{{ objData.precio|int }}
  • Jinja se encarga de insertar los datos directamente en el HTML.
  • La vista solo se dedica a mostrar información, no a procesarla.
  • Mantiene una clara separación entre lógica (backend) y presentación (frontend).
  • Incluye un botón "Continuar" que redirige nuevamente al /index, permitiendo volver a seleccionar otro notebook.
PHP
  • Es un archivo PHP que mezcla HTML y código PHP.
  • Los datos se obtienen directamente desde el objeto.
<?php echo $obj->getMarca(); ?>
<?php echo $obj->getCodigo(); ?>
<?php echo number_format($obj->getPrecio()); ?>
  • El procesamiento y la visualización están más mezclados.
  • PHP ejecuta el código dentro del HTML en tiempo de ejecución.
  • La vista tiene más control directo sobre los datos.
  • Incluye un botón "Continuar" que redirige a index.php, reiniciando el flujo del sistema.

12 Navegación hacia el Formulario de Login

PHP

El enlace apunta directamente a un archivo PHP.

<a href="FormularioLogin.php" class="btn btn-principal">Ir al Login</a>

Al hacer clic: El servidor ejecuta FormularioLogin.php → Se muestra el formulario de login.

Python

El enlace apunta a una ruta del backend.

<a href="/FormularioLogin" class="btn btn-principal">Ir al Login</a>

Al hacer clic: El navegador solicita la ruta /FormularioLogin → Flask ejecuta la función asociada → Se renderiza el formulario de login.

13 Carga del Formulario de Login

Python

  • Flask ejecuta la función asociada a la ruta.
  • Se recuperan los datos desde la sesión.
  • Se renderiza la vista FormularioLogin.html usando Jinja.
  • El backend controla completamente qué se muestra.

  • Se accede directamente al archivo FormularioLogin.php.
  • Se inicia la sesión con session_start().
  • Se obtienen los datos desde $_SESSION.
  • Se incluye la plantilla del formulario de login.
  • El archivo PHP se ejecuta al ser solicitado.

14 Formulario de Login

Python
  • Es una plantilla HTML con Jinja.
  • El formulario envía los datos mediante POST a la ruta /validar.
  • La lista de usuarios se genera dinámicamente usando Jinja.
  • Los datos (objData) llegan desde el backend a través de render_template.
  • El HTML se encarga únicamente de mostrar el formulario, no de procesar la información.
  • Al presionar LOGIN, el flujo continúa hacia la validación en el backend.
PHP
  • Es un archivo PHP que mezcla HTML y código PHP.
  • El formulario envía los datos mediante POST al archivo validar.php.
  • La lista de usuarios se genera usando código PHP dentro del HTML.
  • El archivo accede directamente a los datos disponibles en el servidor.
  • Al presionar LOGIN, se ejecuta el archivo validar.php para procesar la información.

15 Validación de Login y Manejo de Sesión

Python
Login
  • El usuario completa el formulario de login.
  • El formulario envía los datos (usuario y clave) mediante POST a la ruta /validar.
  • Flask recibe los datos usando request.form.
  • Se ejecuta una consulta a la base de datos para verificar si el usuario existe.
  • Si los datos son correctos: El login se considera válido → El usuario puede continuar al sistema.
  • Si los datos son incorrectos: Se renderiza una vista de error.
Sesión
  • Flask utiliza el objeto session.
  • La sesión se guarda en el servidor usando una clave secreta (secret_key).
  • Se almacenan datos importantes, por ejemplo:
session['listaNote'] = notebooks

Estos datos quedan disponibles en todas las rutas mientras la sesión esté activa. La sesión permite mantener información entre páginas sin volver a consultar la base de datos.

PHP
Login
  • El usuario completa el formulario de login.
  • El formulario envía los datos mediante POST al archivo validar.php.
  • PHP recibe los datos usando $_POST.
  • Se ejecuta una consulta SQL para verificar usuario y contraseña.
  • Si los datos son correctos: El login se considera válido → Se redirige al siguiente archivo del sistema.
  • Si los datos son incorrectos: Se incluye directamente una vista de error.
Sesión
  • PHP maneja sesiones con session_start().
  • Los datos se guardan en la superglobal $_SESSION.
$_SESSION['listaNote'] = $notebooks;

La sesión permanece activa mientras el navegador esté abierto o hasta que se cierre explícitamente. Permite compartir datos entre distintos archivos PHP.

16 Manejo de Error de Autenticación

Python
  • Es una plantilla HTML con Jinja.
  • Se muestra cuando el login falla en la ruta /validar.
  • El backend decide mostrar esta vista usando render_template.
  • Usa url_for para cargar correctamente CSS e imágenes.
  • Muestra un mensaje indicando que el usuario o la contraseña son incorrectos.
<a href="/FormularioLogin">Intentar de Nuevo</a>

Permite volver al login sin perder el flujo del sistema.

PHP
  • Es un archivo PHP con HTML.
  • Se muestra cuando la validación falla en validar.php.
  • El archivo de validación incluye directamente esta vista.
include __DIR__ . '/../frontend/templates/ErrorAutenticacion.php';
  • Los recursos se cargan usando rutas directas: <img src="../img/logo.png">
  • Muestra el mismo mensaje de error al usuario.
<a href="FormularioLogin.php">Intentar de Nuevo</a>

Reinicia el flujo del login desde el archivo correspondiente.

17 Visualización del Notebook Seleccionado

Python
  • Se define una ruta llamada /verNotebook que acepta GET y POST.
  • Flask ejecuta la función asociada a la ruta.
  • Se recupera la lista de notebooks desde la sesión.
  • Se obtiene el valor enviado por el formulario (op) usando:
    • request.form si es POST
    • request.args si es GET
  • Se selecciona el notebook correspondiente a partir de la sesión.
  • Finalmente, se renderiza la vista verNotebook.html enviando el objeto seleccionado.
PHP
  • Se accede directamente al archivo verNotebook.php.
  • Se inicia la sesión con session_start().
  • Se obtiene la lista de notebooks desde $_SESSION.
  • Se lee el valor enviado por el formulario (op) usando $_POST o $_GET.
  • Se selecciona el notebook correspondiente.
  • Se incluye directamente la plantilla del frontend.
