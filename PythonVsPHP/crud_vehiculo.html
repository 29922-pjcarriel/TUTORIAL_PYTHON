Flujo de Operaciones: Escritura y Gestión de Archivos (CRUD)

Diagrama completo del flujo de operaciones de escritura: desde la detección de la solicitud POST hasta la persistencia en base de datos y el manejo de archivos de imagen.

Detección y Recepción

El sistema valida la existencia del botón "Guardar" en $_POST (PHP) o request.form (Python).

Enrutamiento por Parámetro 'op'

Un campo oculto distingue si la acción es una inserción ("new") o una actualización ("update").

Base de Datos del Servidor

Destino final de los datos: INSERT para nuevos registros o UPDATE para modificaciones existentes.

Registro Final (INSERT)
  • Datos Textuales y Archivo: Se recogen todos los campos del formulario POST.
  • Consulta INSERT: Se guardan los datos textuales y el nombre del archivo de imagen mediante una consulta INSERT.
  • Solicitud de carga: El archivo se sube al servidor en una ruta específica.
Actualización de Datos (UPDATE)
  • Recuperación de Valores: Se recuperan valores modificados y se prepara la sentencia SQL.
  • Sentencia SQL Dinámica: Se ejecuta una sentencia dinámica filtrada por el ID.
  • Gestión del Flujo Final: Tras la operación, el sistema redirige al usuario o renderiza la lista principal.
