Conexión a MySQL

Comparativa de instalación, configuración de constantes, función de conexión y ejecución de consultas SQL entre PHP y Python.

1 Instalación Previa

PHP

PHP ya incluye el módulo mysqli de forma nativa. No se necesita instalar librerías externas.

// No requiere instalación adicional.
// La extensión mysqli viene con PHP.
Python

Es necesario instalar el conector MySQL mediante pip:

# En terminal / consola
pip install mysql-connector-python

Luego importar en el archivo:

import mysql.connector

2 Definición de Constantes

Ambos lenguajes definen las credenciales de conexión en un archivo separado.

PHP — constantes.php 
<?php
define("SERVER", "localhost");
define("USER",   "root");
define("PASS",   "");
define("BD",     "matriculacionfinal");
?>
Python — constantes.py 
SERVER = "localhost"
USER   = "root"
PASS   = ""
BD     = "matriculacionfinal"
PORT   = 3306

PHP usa define() para crear constantes globales. Python simplemente usa variables en mayúsculas por convención (no son inmutables, pero se tratan como constantes).

3 Acceso a la Configuración

PHP 
include_once("constantes.php");

Después de incluir el archivo, las constantes están disponibles globalmente: SERVER, USER, PASS, BD.

Python 
from constantes import SERVER, USER, PASS, BD

Se importan solo las variables necesarias del módulo.

4 Función Conexión

PHP crea la conexión instanciando la clase nativa mysqli, Python crea la conexión llamando a una función del módulo mysql.connector.

Diferencia clave: PHP usa una clase para la conexión, mientras que Python utiliza una función del módulo importado. Ambos requieren manejo de errores para evitar fallos en la aplicación.

PHP 
function conectar() {
    $c = new mysqli(SERVER, USER, PASS, BD);
    if ($c->connect_error) {
        die("Error: " . $c->connect_error);
    }
    return $c;
}

Uso:

$cn  = conectar();
$obj = new matricula($cn);
Python 
import sys
import mysql.connector

def conectar():
    try:
        c = mysql.connector.connect(
            host     = SERVER,
            user     = USER,
            password = PASS,
            database = BD
        )
        return c
    except mysql.connector.Error as e:
        sys.exit(f"Error: {e}")

Uso:

cn = conectar()
objetoVehiculo = Vehiculo(cn)

5 Definición y Ejecución de Consulta SQL

Una vez establecida la conexión, es necesario definir y ejecutar las consultas SQL para interactuar con la base de datos. Ambos lenguajes permiten escribir consultas SQL de forma similar, pero difieren en la sintaxis de strings y en el método de ejecución.

Definición del SQL

PHP usa strings normales para el SQL, Python usa strings multilínea que mejoran la legibilidad.

PHP 
$sql = "SELECT v.id, v.placa,
    m.descripcion as marca,
    mo.descripcion as modelo,
    v.anio, v.costo, v.foto
    FROM vehiculo v
    INNER JOIN marca m
    ON v.id_marca = m.id
    INNER JOIN modelo mo
    ON v.id_modelo = mo.id";
Python 
sql = """SELECT v.id, v.placa,
    m.descripcion as marca,
    mo.descripcion as modelo,
    v.anio, v.costo, v.foto
    FROM vehiculo v
    INNER JOIN marca m
    ON v.id_marca = m.id
    INNER JOIN modelo mo
    ON v.id_modelo = mo.id"""

Python usa comillas triples (""") para strings multilínea, similar a las comillas dobles de PHP pero sin necesidad de concatenar líneas.

Ejecución de la Consulta

PHP ejecuta el SQL directamente desde el objeto conexión, Python requiere un cursor para ejecutar consultas.

PHP
  • PHP ejecuta el SQL directamente desde el objeto conexión.
  • El método query() ejecuta la consulta y devuelve el resultado.
  • No requiere crear un cursor adicional.
$res = $this->con->query($sql);

Devuelve un objeto mysqli_result que se recorre con fetch_assoc().

Python
  • Python requiere un cursor para ejecutar consultas.
  • El cursor actúa como intermediario entre la conexión y la base de datos.
  • Permite configurar cómo se devuelven los resultados.
cursor = self.__con.cursor(dictionary=True)
cursor.execute(sql)

El parámetro dictionary=True devuelve cada fila como un diccionario (clave → nombre de columna).

6 ¿Cómo Construimos HTML Dinámico?

Al construir HTML dinámico, es necesario insertar variables en cadenas de texto. PHP y Python tienen diferentes estrategias para lograr esto.

PHP - Concatenación

Se insertan variables en cadenas usando concatenación con el operador . (punto) cuando se concatenan usando comillas dobles "".

// Ejemplo de concatenación en PHP
$html = "";
$html .= "<html>";
$html .= "  <div class='row'>";

// Insertar variables directamente
<input type="hidden" name="id" value="$id">
<input type="hidden" name="op" value="$op">

$html .= "  </div>";
$html .= "</html>";

return $html;
Python - F-Strings

Una f-string es una cadena de texto en Python que permite insertar variables o expresiones directamente dentro de la cadena, usando f"" y llaves {}.

# Ejemplo de f-strings en Python
html = ""
html += "<html>"
html += "  <div class='row'>"

# Insertar variables con f-strings
html += f'<input type="hidden" name="id" value="{id}">'
html += f'<input type="hidden" name="op" value="{op}">'

html += "  </div>"
html += "</html>"

return html

Diferencia clave: PHP usa concatenación con el operador . mientras que Python usa f-strings con f"" y llaves {} para una sintaxis más limpia y legible.

7 Manejo del GET y POST

Cuando un usuario llena un formulario en una página web, los datos deben enviarse al servidor. PHP y Python permiten recibir esos datos usando dos métodos principales: GET y POST.

GET
  • Envía los datos por la URL.
  • Visible en la barra de direcciones.
  • Ideal para búsquedas o información que no sea sensible.
POST
  • Envía los datos por el cuerpo de la solicitud.
  • No visible en la URL.
  • Ideal para contraseñas, registros, formularios de contacto.

8 Punto de Entrada de la Aplicación

PHP

No necesita definir rutas explícitas para recibir las solicitudes.

Cuando alguien abre index.php?=xxx, PHP automáticamente pone los datos de GET en $_GET y los de POST en $_POST.

En PHP basta con acceder a las variables globales $_GET y $_POST. En Python (Flask) necesitas definir rutas y usar request para acceder a los datos.

Python

Necesita definir rutas usando @app.route("/").

Para recibir datos, usamos request.args para GET y request.form para POST.

from flask import Flask, request

@app.route("/", methods=["GET", "POST"])
request.args:  # GET
request.form:  # POST

9 ¿Cómo se Captura GET?

Los datos GET viajan en la URL. Ambos lenguajes decodifican el parámetro d en base64 y extraen la operación y el id para determinar qué acción ejecutar.

PHP — index.php 
if(isset($_GET["d"])){
    echo "<br>PETICIÓN GET <br>";
    echo "<pre>";
    print_r($_GET);
    echo "</pre>";

    // 2.2 DETALLE id
    $dato = base64_decode($_GET["d"]);
    $tmp  = explode("/", $dato);

    $op = $tmp[0];
    $id = $tmp[1];

    if($op == "det"){
        echo $v->get_detall_vehiculo($id);
    }elseif($op == "act"){
        echo $v->get_form($id);
    }elseif($op == "new"){
        echo $v->get_form();
    }elseif($op == "del"){
        echo $v->delete_vehiculo($id);
    }
}
Python — app.py 
if 'd' in request.args:
    contenido += "<br>PETICIÓN GET <br>"
    contenido += "<pre>"
    contenido += str(dict(request.args))
    contenido += "</pre>"

    # 2.2 DETALLE id
    dato = base64.b64decode(request.args.get("d")).decode("utf-8")
    tmp  = dato.split("/")

    op = tmp[0]
    id = tmp[1]

    if op == "det":
        contenido += v.get_detail_vehiculo(id)
    elif op == "act":
        contenido += v.get_form(id)
    elif op == "new":
        contenido += v.get_form()
    elif op == "del":
        contenido += v.delete_vehiculo(id)

En ambos lenguajes, el parámetro d llega codificado en base64. Se decodifica y se separa con / para obtener la operación (det, act, new, del) y el id del registro.

10 ¿Cómo se Captura POST? — Formulario HTML

El formulario HTML utiliza el método POST con enctype="multipart/form-data" para enviar datos al servidor, incluyendo archivos como fotos.

PHP — class.vehiculo.php 
// Formulario con acción a index.php
<form name="Form_vehiculo"
      method="POST"
      action="index.php"
      enctype="multipart/form-data">
  <input type="hidden" name="id"
         value="' . $id . '">
  <input type="hidden" name="op"
         value="' . $op . '">

  // Campo de archivo
  <div class="col-sm-4 form-group">
    <label for="foto">Foto</label>
    <input type="file" name="foto" ' . $flag . '>
  </div>

  // Botón guardar
  <button type="submit" name="Guardar"
          class="btn btn-primary">
    <i class="fa fa-save"></i> Guardar
  </button>
</form>
Python — vehiculo.py 
# Formulario con acción a ruta "/"
<form name="Form_vehiculo"
      method="POST"
      action="/"
      enctype="multipart/form-data">
  <input type="hidden" name="id"
         value="{id}">
  <input type="hidden" name="op"
         value="{op}">

  # Campo de archivo
  <div class="col-sm-4 form-group">
    <label for="foto">Foto</label>
    <input type="file" name="foto" {flag}>
  </div>

  # Botón guardar
  <button type="submit" name="Guardar"
          class="btn btn-primary">
    <i class="fa fa-save"></i> Guardar
  </button>
</form>

Diferencia clave: PHP usa concatenación (. $id .) para insertar variables en el HTML, mientras que Python usa f-strings ({id}). El formulario PHP apunta a index.php, el de Python apunta a la ruta /.

12 Mensajes de Errores

Ambos lenguajes generan HTML dinámico para mostrar alertas de error al usuario, utilizando Bootstrap para el estilo visual.

PHP — class.vehiculo.php 
private function message_error($tipo){
    $html = ''
    . '<div class="row">' . "\n"
    . '  <div class="col-md-8 col-md-offset-2">' . "\n"
    . '    <div class="alert alert-danger
              alert-dismissible" role="alert">' . "\n"
    . '      <button type="button" class="close"
                data-dismiss="alert"
                aria-label="Close">' . "\n"
    . '        <span aria-hidden="true">
                &times;</span>' . "\n"
    . '      </button>' . "\n"
    . '      <i class="fa fa-exclamation-circle">
            </i> <strong>Error:</strong>
            Error al ' . $tipo . ' <br>' . "\n"
    . '      Favor contactar a administrador.' . "\n"
    . '    </div>' . "\n"
    . '    <div style="text-align: center;
              margin-top: 20px;">' . "\n"
    . '      <a href="index.php"
              class="btn btn-default">' . "\n"
    . '        <i class="fa fa-arrow-left">
                </i> Regresar' . "\n"
    . '      </a>' . "\n"
    . '    </div>' . "\n"
    . '  </div>' . "\n"
    . '</div>';
    return $html;
}
Python — vehiculo.py 
def __message_error(self, tipo):

    return f'''
    <div class="alert alert-danger

      <button type="button" class="close"

              &times;</span>
      </button>
      <strong><i class="fa fa-exclamation-circle">
      </i> Error:</strong> Al {tipo}.

         margin-top: 20px;">
      <a href="/" class="btn btn-primary">
        <i class="fa fa-arrow-left"></i>
        Regresar
      </a>
    </div>
    '''

PHP construye el HTML con concatenación (.) y saltos de línea (\n). Python usa f-strings con triple comilla (f'''...''') para escribir el HTML multilínea de forma más legible, insertando la variable {tipo} directamente.
