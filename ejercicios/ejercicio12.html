Tablas dinámicas

Implementación de tablas dinámicas.

¿Qué son las tablas dinámicas?

Una tabla dinámica es una tabla que no se construye de forma fija, sino que se genera automáticamente a partir de los datos contenidos en arreglos o matrices.

El número de filas, columnas y celdas depende directamente de la información almacenada en las estructuras de datos. 

tabla[filas][columnas] generadas dinámicamente

Enunciado

Desarrollar un programa que represente países, provincias y ciudades mediante arreglos multidimensionales y genere dinámicamente tablas HTML que muestren esta información organizada por país.

Objetivo: Comprender cómo los arreglos permiten construir tablas dinámicas sin definir previamente el número de filas y columnas.

¿Por qué este ejercicio es de tablas dinámicas?

  • Las provincias determinan automáticamente el número de columnas
  • Las ciudades determinan automáticamente el número de filas
  • La tabla se adapta a cualquier cantidad de datos
Columnas = número de provincias
Filas    = máximo de ciudades

Recorrido de los datos

Para construir la tabla se utilizan recorridos anidados:

for pais:
    for provincia:
        for ciudad:
            generar celda

Este tipo de recorrido permite construir tablas completamente adaptables a la información disponible.

Explicación detallada del código (párrafo en párrafo)

1) Importación de utilidades Flask

from flask import request, render_template_string importa dos piezas clave para que el ejercicio funcione dentro de tu plataforma. request te permite saber si el usuario solo está viendo la página (GET) o si presiona el botón Ejecutar ejercicio (POST). render_template_string sirve para devolver HTML construido desde un string en Python, sin necesidad de crear un archivo HTML extra para el resultado. 

from flask import request, render_template_string
2) Encabezado del ejercicio

Este bloque sirve para identificar el ejercicio, su temática y que fue convertido desde PHP a Python. No cambia la ejecución, pero ayuda a mantener organizado el proyecto y entender rápidamente el objetivo del archivo. 

# =====================================================
# EJERCICIO: TABLAS DINÁMICAS - PROVINCIAS
# CONVERSIÓN 1:1 DESDE PHP A PYTHON
# =====================================================
3) Plantilla HTML base (variable HTML)

HTML guarda una plantilla en texto que luego Flask renderiza. Allá­ incluyes estilos y contenido inicial. En GET, se muestra el mensaje para ejecutar. En POST, se muestran las tablas generadas (cuando la plantilla está preparada para recibir tablas y mostrarlas). 

HTML = """ ... """
4) CSS para tablas y celdas

Se aplican estilos para que las tablas queden centradas, con fondo blanco, bordes limpios y texto alineado. Como las tablas se generan por código, este CSS asegura que todas se vean iguales sin importar cuántas columnas o filas existan. 

table{
  margin:auto;
  background:white;
  border-collapse: collapse;
}
td, th{
  padding:6px 10px;
  border:1px solid #333;
  text-align:center;
}
5) Capital resaltada con parpadeo (blink)

En cada provincia, la primera ciudad (posición 0) se considera la capital. Para resaltarla se aplica la clase blink que usa una animación CSS. Así­ el texto alterna visibilidad y llama la atención, igual que en tu PHP original. 

.blink{
  animation: blink 1s linear infinite;
  font-weight:bold;
  color:black;
}
@keyframes blink {
  50% { opacity:0; }
}
6) Evitar desbordes (scroll horizontal)

Como el ejercicio puede tener muchas provincias (muchas columnas), en pantallas pequeñas la tabla podría salirse del contenedor. Con overflow-x:auto se habilita un scroll horizontal para mantener el diseño estable. 

.table-wrap{
  width:100%;
  overflow-x:auto;
}
7) Control GET vs POST dentro de ejecutar()

La función ejecutar() es llamada por tu app.py. Primero se valida el método HTTP: si es GET, no se genera nada y solo se muestra el mensaje inicial. Si es POST, se construyen tablas y se retorna el resultado. 

def ejecutar():

    if request.method == "GET":
        return render_template_string(HTML, mostrar=False)
8) Estructura principal paises

paises es un diccionario anidado (3 niveles): país -> provincia -> lista de ciudades. Esta estructura permite construir tablas dinámicas porque el número de provincias define las columnas y el máximo número de ciudades define las filas. 

paises = {
  "ECUADOR": {
    "Pichincha": ["QUITO", "..."],
    "Guayas": ["GUAYAQUIL", "..."]
  }
}
9) Generación dinámica de tablas

Se recorre cada país y se genera una tabla HTML. Primero se calcula maxFilas con el máximo tamaño de listas (para que todas las columnas tengan la misma cantidad de filas). Luego se construye el encabezado con provincias y el cuerpo recorriendo fila por fila. 

tablas = []

for pais, infoPais in paises.items():

    maxFilas = max(len(c) for c in infoPais.values())

    html = f"<br><strong>PAÍS: {pais}</strong><br>"
    html += "<table>"
    html += f"<tr><th colspan='{len(infoPais)}' bgcolor='#EC7063'>{pais}</th></tr><tr>"

    for prov in infoPais:
        html += f"<th>{prov}</th>"
    html += "</tr>"
10) Cuerpo de la tabla (capital + celdas vacías)

En la fila 0 se imprime la capital con blink. En las demás filas se imprime la ciudad si existe. Si no existe (provincia con menos ciudades), se agrega una celda vacía para mantener alineadas las columnas. 

for i in range(maxFilas):
    html += "<tr>"
    for cantones in infoPais.values():
        if i == 0:
            html += f"<td class='blink' bgcolor='#D6FAF2'>{cantones[0]}</td>"
        else:
            html += (
                f"<td bgcolor='#D6FAF2'>{cantones[i]}</td>"
                if i < len(cantones)
                else "<td bgcolor='#D6DEFA'> </td>"
            )
    html += "</tr>"
11) Render final (enviar tablas a la vista)

Se devuelve la plantilla renderizada y se envían las variables que la vista necesita: mostrar para controlar la presentación y tablas para imprimir los HTML generados. Para que se vean como HTML real, en la plantilla debes imprimirlas usando |safe. 

return render_template_string(
    HTML,
    mostrar=True,
    paises=paises,
    tablas=tablas
)

Código completo del ejercicio (Python)

Este es el código completo del ejercicio convertido desde PHP a Python, manteniendo la misma estructura de datos, recorridos y comportamiento visual. Analice el código antes de ejecutarlo. 

from flask import Flask, request, render_template_string

app = Flask(__name__)

# =====================================================
# MATRIZ PRINCIPAL
# PAÍS - PROVINCIA - CANTONES / CIUDADES
# =====================================================
PAISES = {
    "ECUADOR": {

        "Pichincha": ["Quito", "Cayambe", "Rumiñahui", "Mejía"],

        "Guayas": ["Guayaquil", "Durán", "Milagro", "Daule"],
        "Azuay": ["Cuenca", "Gualaceo", "Paute"]
    },
    "COLOMBIA": {
        "Cundinamarca": ["Bogotá", "Soacha"],
        "Antioquia": ["Medellín", "Envigado", "Rionegro"],
        "Valle del Cauca": ["Cali", "Palmira", "Yumbo"]
    },
    "USA": {
        "California": ["Los Angeles", "San Diego", "San Francisco"],
        "Texas": ["Houston", "Dallas", "Austin"],
        "Florida": ["Miami", "Orlando", "Tampa", "Jacksonville"]
    }
}

# =====================================================
# TEMPLATE HTML (INLINE PARA FINES ACADÉMICOS)
# =====================================================
HTML = """
<!DOCTYPE html>
<html lang="es">
<head>
<meta charset="UTF-8">
<title>Tablas Dinámicas - Flask</title>
<style>
body{
  font-family: Arial, sans-serif;
  background:#f4f6f7;
}
table{
  margin:auto;
  border-collapse:collapse;
  background:white;
}
th, td{
  border:1px solid #333;
  padding:8px 12px;
  text-align:center;
}
th{
  background:#2e86c1;
  color:white;
}
.blink{
  animation: blink 1s linear infinite;
  font-weight:bold;
}
@keyframes blink{
  50%{opacity:0;}
}
.container{
  width:90%;
  margin:auto;
  text-align:center;
}
</style>
</head>

<body>
<div class="container">

<h2>TABLAS DINÁMICAS - PROVINCIAS</h2>

<form method="POST">
  <label><strong>Seleccione un país:</strong></label><br><br>
  <select name="pais" required>
    <option value="">-- Seleccione --</option>
    {% for p in paises %}
      <option value="{{p}}" {% if p == pais_seleccionado %}selected{% endif %}>
        {{p}}
      </option>
    {% endfor %}
  </select>
  <br><br>
  <button type="submit">Mostrar Tabla</button>
</form>

{% if tabla %}
  <hr>
  {{ tabla|safe }}
{% endif %}

</div>
</body>
</html>
"""

# =====================================================
# RUTA PRINCIPAL
# =====================================================
@app.route("/", methods=["GET", "POST"])
def ejecutar():

    tabla_html = None
    pais_seleccionado = None

    # -----------------------------
    # POST EJECUCIÓN DINÁMICA
    # -----------------------------
    if request.method == "POST":

        pais_seleccionado = request.form.get("pais")
        provincias = PAISES[pais_seleccionado]

        max_filas = max(len(cantones) for cantones in provincias.values())

        tabla = f"<h3>País: {pais_seleccionado}</h3>"
        tabla += "<table>"
        tabla += "<tr>"

        for provincia in provincias:
            tabla += f"<th>{provincia}</th>"
        tabla += "</tr>"

        for i in range(max_filas):
            tabla += "<tr>"
            for cantones in provincias.values():
                if i < len(cantones):
                    clase = "blink" if i == 0 else ""
                    tabla += f"<td class='{clase}'>{cantones[i]}</td>"
                else:
                    tabla += "<td> </td>"
            tabla += "</tr>"

        tabla += "</table>"

        tabla_html = tabla

    return render_template_string(
        HTML,
        paises=PAISES.keys(),
        tabla=tabla_html,
        pais_seleccionado=pais_seleccionado
    )

# =====================================================
# EJECUCIÓN
# =====================================================
if __name__ == "__main__":
    app.run(debug=True)
