Procesamiento de formularios

Formulario request.files

Formulario POST con subida de archivos (files).

Enunciado

En este ejercicio vamos a crear un formulario de registro vehicular que envía datos al servidor usando el método POST. Además, el usuario podrá subir una imagen (foto del vehículo) desde el navegador.

Objetivo: entender cómo Flask recibe datos con request.form y cómo recibe archivos con request.files (equivalente a $_POST y $_FILES en PHP).

Explicación del código (paso a paso)

0) Idea general (GET vs POST)
  • GET: el usuario entra a la página -> solo mostramos el formulario vacío.
  • POST: el usuario llena el formulario y presiona Enviar -> el servidor procesa los datos y el archivo.
  • En Flask detectamos esto con request.method.
1) Importaciones
import os
import random
import string
from flask import request, render_template_string
  • os: crear rutas y carpetas para guardar la imagen.
  • random y string: generar nombres aleatorios para archivos.
  • request: contiene lo que el navegador envía (POST, FILES, etc.).
  • render_template_string: renderiza HTML guardado en una variable.
< 
BASE_DIR = os.path.dirname(os.path.abspath(__file__))
UPLOAD_FOLDER = os.path.join("static", "images")
os.makedirs(UPLOAD_FOLDER, exist_ok=True)

Se define una carpeta para subir imÃ¡genes (static/images). Luego se crea si no existe. Lo recomendado es usar una ruta absoluta con BASE_DIR para evitar errores al guardar.

3) Crear un nombre aleatorio (evitar duplicados)
def get_name_file(nombre_original, tamanio):
    ext = nombre_original.split('.')[-1]
    letras = string.ascii_letters
    nombre = ''.join(random.choice(letras) for _ in range(tamanio))
    return f"{nombre}.{ext}"
  • Se obtiene la extensiÃ³n (jpg, png, etc.).
  • Se genera un nombre aleatorio para no repetir archivos.
  • Se devuelve: nombre_aleatorio.ext.
4) POST + FILES en Flask (comparaciÃ³n con PHP)

En PHP usabas $_POST para datos de texto y $_FILES para archivos. En Flask se usa el objeto request:

PHP
$_POST["placa"]
$_FILES["foto"]["name"]

PHP separa datos y archivos en superglobales.

Flask (Python)
request.form.get("placa")
request.files.get("foto")

Flask usa request.form y request.files.

5) Por qué es obligatorio: multipart/form-data

Para que el navegador envíe un archivo correctamente, el formulario debe tener: 

<form method="post" enctype="multipart/form-data">
Si no colocas multipart/form-data, request.files llegará vacío aunque selecciones una imagen.
6) Detectar POST y leer los datos del formulario
if request.method == 'POST':
    placa = request.form.get('placa')
    marca = request.form.get('marca')
    combustible = request.form.get('combustible')
    comentarios = request.form.get('comentarios')
    accesorios = request.form.get('accesorio')

Todo lo que NO es archivo (input, select, textarea, radio) llega en request.form. Usamos .get() para recuperar valores de forma segura.

7) Leer el archivo con request.files
foto = request.files.get('foto')
foto_url = None

Aquí Flask busca el archivo cuyo input tiene name="foto". Si el usuario no sube nada, foto puede venir vacío.

8) Guardar la imagen en el servidor
if foto and foto.filename:
    nombre = get_name_file(foto.filename, 15)
    ruta = os.path.join(UPLOAD_FOLDER, nombre)
    foto.save(ruta)
    foto_url = f"/static/assets/img/{nombre}"
  • foto.filename verifica que sí se eligió un archivo.
  • foto.save(...) guarda el archivo en tu proyecto.
  • foto_url es la ruta web que luego se usa en el HTML para mostrar la imagen.
9) Renderizar el resultado (mostrar datos + imagen)
return render_template_string(
    HTML,
    placa=placa,
    marca=marca,
    combustible=combustible,
    accesorios=accesorios,
    comentarios=comentarios,
    foto=foto_url,
    mostrar=True
)

Se vuelve a renderizar el HTML, pero ahora con mostrar=True y con variables llenas para que se vea el resultado.
