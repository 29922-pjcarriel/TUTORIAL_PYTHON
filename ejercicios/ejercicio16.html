Interfaces

Ejercicio de Protocolos y contratos de software.

¿Qué son las Interfaces en Python?

A diferencia de otros lenguajes, en Python las interfaces se implementan mediante Protocolos (typing.Protocol). Un protocolo define un contrato: cualquier clase que implemente los métodos definidos en el protocolo se considera que cumple con la interfaz, permitiendo el tipado estructural o "Duck Typing" estático.

Enunciado

Definir un protocolo llamado Formulas que exija la implementación de los métodos area() y perimetro(). Posteriormente, crear clases como Cuadrado, Triangulo y Rombo que cumplan con dicho contrato y validar su cumplimiento en tiempo de ejecución.

¿Por qué este ejercicio aplica Interfaces?

  • Se define un Protocolo como contrato de métodos.
  • Se utiliza el decorador @runtime_checkable para validaciones dinámicas.
  • Las clases no necesitan heredar explícitamente de la interfaz para cumplirla (Tipado Estructural).
  • Se garantiza que cualquier figura procesada tenga los métodos de cálculo necesarios.

Explicación detallada del código

1) Importación de utilidades de tipado

Para definir interfaces en Python moderno, importamos Protocol y runtime_checkable desde el módulo typing. Estas herramientas permiten definir contratos que pueden ser validados durante la ejecución del programa. 

from typing import Protocol, runtime_checkable, Any
from flask import request
2) Definición del Protocolo (Interfaz)

Creamos la clase Formulas heredando de Protocol. A diferencia de una clase normal, esta no implementa lógica, sino que define la "forma" que deben tener las clases que quieran ser consideradas figuras válidas. 

class Formulas(Protocol):
3) Decorador @runtime_checkable

Este decorador es fundamental, ya que permite que podamos usar la función estándar isinstance() para verificar si un objeto cumple con el protocolo en tiempo de ejecución, facilitando la validación de los contratos. 

@runtime_checkable
class Formulas(Protocol):
4) Definición de firmas de métodos

Dentro del protocolo, definimos los métodos area() y perimetro(). Usamos puntos suspensivos (...) para indicar que solo estamos definiendo la firma y el tipo de retorno, dejando la implementación a las clases reales. 

    def area(self) -> Any:
        ...
    def perimetro(self) -> Any:
        ...
5) Implementación en la clase Cuadrado

La clase Cuadrado implementa los métodos requeridos. Observe que no necesita heredar explícitamente de Formulas; con el simple hecho de tener los mismos métodos, Python reconoce que cumple el contrato. 

class Cuadrado:
    def __init__(self, lado):
        self.lado = lado
    def area(self): return self.lado ** 2
    def perimetro(self): return self.lado * 4
6) Cumplimiento del contrato en Triangulo

De igual forma, la clase Triangulo implementa sus propias fórmulas. Al tener definidos area() y perimetro(), se vuelve automáticamente compatible con cualquier función que espere un objeto Formulas. 

class Triangulo:
    def __init__(self, l1, l2, l3):
        # ... inicialización ...
    def area(self): # ... lógica Herón ...
    def perimetro(self): return self.l1 + self.l2 + self.l3
7) Flexibilidad con la clase Rombo

El uso de interfaces permite que figuras con propiedades muy distintas, como el Rombo (que usa diagonales), puedan ser tratadas de forma uniforme siempre que respeten los nombres de los métodos del protocolo. 

class Rombo:
    def __init__(self, d_mayor, d_menor):
        self.D, self.d = d_mayor, d_menor
    def area(self): return (self.D * self.d) / 2
8) Validación de tipos con isinstance()

En la lógica de ejecución, creamos el objeto y luego verificamos si cumple con la interfaz usando isinstance(objeto, Formulas). Esto asegura que el programa no falle al intentar llamar a los métodos de cálculo. 

if isinstance(figura, Formulas):
    area = figura.area()
    perimetro = figura.perimetro()
9) Captura dinámica de parámetros

Dependiendo de la figura seleccionada por el usuario en el formulario, el sistema extrae los valores necesarios (Lado, Base, Diagonales) para instanciar la clase correspondiente antes de validarla contra la interfaz. 

val1 = float(request.form.get("valor1", 0))
val2 = float(request.form.get("valor2", 0))
# ... lógica de instanciación ...
10) Generación de la respuesta final

Si el objeto cumple el contrato, se formatea un bloque HTML con los resultados. Si no lo cumpliera (por ejemplo, si faltara un método), se informaría al usuario que la clase no respeta el protocolo definido. 

return f"""
<div class='alert alert-info'>
  <p><strong>Contrato 'Formulas':</strong> CUMPLIDO</p>
  <p><strong>Área:</strong> {area:.2f}</p>
</div>
"""

CÓDIGO COMPLETO DEL EJERCICIO (Clase05_interfaces.py)

Este bloque contiene la implementación total de Interfaces usando Protocolos. Incluye la definición del contrato y tres implementaciones estructurales. 

from typing import Protocol, runtime_checkable, Any
import math
from flask import Flask, request

app = Flask(__name__)

# =====================================================
# INTERFAZ (PROTOCOLO)
# =====================================================
@runtime_checkable
class Formulas(Protocol):
    def area(self) -> Any:
        ...

    def perimetro(self) -> Any:
        ...

# =====================================================
# IMPLEMENTACIONES (CLASES)
# =====================================================
class Cuadrado:
    def __init__(self, lado):
        self.lado = float(lado)

    def area(self):
        return self.lado * self.lado

    def perimetro(self):
        return self.lado * 4

class Triangulo:
    def __init__(self, l1, l2, l3):
        self.l1, self.l2, self.l3 = float(l1), float(l2), float(l3)

    def perimetro(self):
        return self.l1 + self.l2 + self.l3

    def area(self):
        s = self.perimetro() / 2
        return math.sqrt(s * (s - self.l1) * (s - self.l2) * (s - self.l3))

class Rombo:
    def __init__(self, diag_mayor, diag_menor):
        self.D = float(diag_mayor)
        self.d = float(diag_menor)

    def area(self):
        return (self.D * self.d) / 2

    def perimetro(self):
        # Perímetro basado en diagonales
        lado = math.sqrt((self.D/2)**2 + (self.d/2)**2)
        return lado * 4

# =====================================================
# RUTA DE EJECUCIÓN
# =====================================================
@app.route("/", methods=["GET", "POST"])
def ejecutar():
    if request.method == "POST":
        fig = request.form.get("figura")
        v1 = float(request.form.get("v1", 0))
        v2 = float(request.form.get("v2", 0))
        v3 = float(request.form.get("v3", 0))

        if fig == "cuadrado":
            objeto = Cuadrado(v1)
        elif fig == "triangulo":
            objeto = Triangulo(v1, v2, v3)
        elif fig == "rombo":
            objeto = Rombo(v1, v2)

        # Validación del Contrato (Interfaz)
        cumple = isinstance(objeto, Formulas)
        
        if cumple:
            a = objeto.area()
            p = objeto.perimetro()
            return f"""
            <div class='alert alert-success'>
              <h4>Validación de Interfaz</h4>
              <p><strong>Protocolo 'Formulas':</strong> CUMPLE ✅</p>
              <p><strong>Área:</strong> {a:.2f}</p>
              <p><strong>Perímetro:</strong> {p:.2f}</p>
            </div>
            """
        else:
            return "<div class='alert alert-danger'>La clase no cumple con el protocolo.</div>"

    return "Esperando POST..."

if __name__ == "__main__":
    app.run(debug=True)
