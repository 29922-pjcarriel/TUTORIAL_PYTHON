Matriz asociativa

Diccionario de provincias con ciudades y tabla dinámica.

Enunciado

Crear una estructura con cantones (índices numéricos) y un diccionario ciudades (matriz asociativa) donde cada provincia tenga una lista de ciudades. Luego calcular cuántas provincias hay, el máximo número de ciudades por provincia y mostrar una tabla con columnas dinámicas.

Objetivo: entender cómo se combinan listas y diccionarios para construir una matriz asociativa y cómo generar una tabla en base al máximo de elementos.

Explicación del código (paso a paso)

1) Índices numéricos: cantones

Primero se define una lista de listas llamada cantones. Cada fila contiene cantones y se accede por posición, por ejemplo: cantones[1][0]. 

const cantones = [
  ["QUITO", "CAYAMBE"],
  ["GUAYAQUIL", "DAULE"],
  ["CUENCA", "GUALACEO"],
  ["PORTOVIEJO", "MANTA", "CHONE", "CRUCITA"]
];
2) Matriz asociativa: ciudades

Luego se arma un diccionario (objeto) donde la clave es la provincia y el valor es una lista de ciudades. En "Guayas" se reutiliza lo que ya existe en cantones para no repetir valores. 

const ciudades = {
  "Pichincha": ["QUITO", "CAYAMBE", "RUMIÑAHUI"],
  "Guayas": [cantones[1][0], cantones[1][1], "PLAYAS"],
  "Azuay": ["Cuenca"],
  "Manabi": cantones[3],
  "Tunguragua": ["Ambato"]
};
3) Cálculos: número de provincias y máximo de columnas

num_provincias es la cantidad de claves del diccionario. max_colum es el tamaño de la lista más larga de ciudades (para saber cuántas columnas crear). 

const num_provincias = Object.keys(ciudades).length;

let max_colum = 0;
for (const provincia in ciudades) {
  max_colum = Math.max(max_colum, ciudades[provincia].length);
}
4) Salida: tabla con columnas dinámicas

Se construye una tabla HTML: una columna para "Provincia" y luego "Ciudad 1..N" según el máximo encontrado. Si una provincia tiene menos ciudades, las celdas quedan vacías.
