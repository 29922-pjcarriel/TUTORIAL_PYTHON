1. Enunciado del Ejercicio (Sesiones)

En este módulo construiremos un sistema de catálogo de computadoras aplicando el concepto de Sesiones HTTP usando Python con Flask como backend.

Importante: Este ejercicio se centra en la persistencia de datos temporal en el servidor. No utilizaremos base de datos en esta etapa, ya que el objetivo es comprender el funcionamiento interno del objeto session de Flask y cómo "recordar" la selección del usuario entre distintas páginas.

En este ejercicio el estudiante aprenderá a:

  • Configurar la clave secreta (Secret Key): Elemento esencial para firmar criptográficamente las cookies de sesión.
  • Gestionar el diccionario session: Almacenar y recuperar valores (en este caso, la marca de una notebook) a través de distintas peticiones.
  • Implementar lógica de navegación: Mantener el estado del usuario consistente mientras navega desde la selección hasta el detalle del producto.
Concepto Clave: Una sesión permite almacenar información específica de un usuario de una solicitud a otra. Se implementa sobre cookies firmadas, lo que garantiza que los datos no puedan ser manipulados por el cliente.

2. Enunciado del Ejercicio (Login de Usuarios)

En este módulo construiremos un sistema de autenticación de usuarios completo, aplicando validación de credenciales usando Python con Flask como backend y MySQL como base de datos.

Importante: Este ejercicio introduce una arquitectura de software más robusta. Separaremos el código en controladores (app.py), vistas (carpetas html) y clases (lógica de negocio), simulando un entorno de desarrollo profesional independiente del ejercicio anterior.

En este ejercicio el estudiante aprenderá a:

  • Conectar Flask con MySQL: Utilizar el conector oficial para realizar consultas a la base de datos sesionesbd.
  • Validar credenciales: Verificar usuario y contraseña mediante consultas SQL seguras.
  • Gestionar el estado de autenticación: Crear y verificar variables de sesión como session['login'] = True para controlar el acceso.
  • Proteger rutas: Implementar lógica que impida el acceso a páginas internas (Dashboard) si el usuario no ha iniciado sesión.
  • Implementar Logout: Cerrar la sesión de forma segura eliminando los datos almacenados.
Estructura Arquitectónica

Organizamos el código en capas para mejorar la mantenibilidad:

|-- app.py              # Controlador Principal (Orquesta el flujo)
|
|-- paginas/            # Capa de Presentación (Vistas HTML)
|   |-- index.html      # Landing Page
|   |-- login.html      # Interfaz de Entrada
|   |-- home.html       # Área Privada (Dashboard)
|   |-- error.html      # Feedback de Error
|
|-- clases11/           # Capa de Negocio (Modelos)
|   |-- notebook.py     # Clase de Objeto
Flujo de Seguridad: El sistema validará credenciales contra MySQL. Si son válidas, activará la sesión. Cada vez que el usuario intente ver una notebook, el sistema verificará primero si la sesión está activa.
