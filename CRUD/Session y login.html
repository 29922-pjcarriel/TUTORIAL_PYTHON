Ejercicio 1: Gestión de Sesiones (Notebooks)

Este primer ejercicio se centra exclusivamente en el manejo de variables de sesión en Flask. No utilizamos base de datos ni validación de usuarios; simplemente seleccionamos un producto y lo mantenemos en la memoria de la sesión.

Objetivo: Seleccionar una marca en la página principal y que el sistema "recuerde" esta selección en la página de detalles, persistiendo la información durante la navegación.
Componentes de este Ejercicio
  • app.py: Script simple que configura session["marca_seleccionada"].
  • templates/index.html: Formulario de selección de marca.
  • templates/verNotebook.html: Vista que recupera la variable session.

Ejercicio 2: Login con MySQL

Este segundo ejercicio es independiente del anterior. Aquí implementamos un sistema completo de Autenticación de Usuarios utilizando una base de datos MySQL (sesionesbd) y una estructura de archivos más organizada.

Estructura de Archivos (Ejercicio Login)

Organizamos el código separando clases, paginas (vistas) y controlador:

|-- app.py              # Controlador Principal (Login + MySQL)
|
|-- paginas/            # Carpeta para las Vistas HTML
|   |-- index.html      # Página de Bienvenida / Menú
|   |-- login.html      # Formulario de Acceso
|   |-- home.html       # Página Principal (Dashboard)
|   |-- error.html      # Pantalla de Error de Login
|
|-- clases11/           # Clases de Negocio
|   |-- notebook.py     # Clase 'Notebook'
Lógica de Funcionamiento
  1. Inicio: El usuario accede a la ruta / o /login para ver el formulario.
  2. Validación (MySQL): Al enviar el formulario a /validar, Flask conecta con la base de datos sesionesbd.
  3. Verificación:
    • Si existe el usuario: Se crea session["usuario"] y session["login"] = True.
    • Si NO existe: Se muestra la vista error.html.
  4. Acceso: Si el login es correcto, el sistema renderiza la vista protegida con los datos del usuario.
