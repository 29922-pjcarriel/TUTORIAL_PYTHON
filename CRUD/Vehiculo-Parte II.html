Módulo CRUD - Paso 02

Vehiculo - Parte II

Detalle y Borrar funcionando + navegación con Matrículas.

Volver a CRUD

1. Enunciado del Ejercicio (Vehículo - Parte II)

En esta Parte II continuamos el sistema de matriculación de vehículos aplicando el enfoque CRUD (Create, Read, Update, Delete) con Python (Flask) y MySQL.

A diferencia de la Parte I (que solo preparaba la base y la interfaz), en esta Parte II el objetivo es que el módulo de Vehículos muestre acciones reales en pantalla, especialmente:

  • Detalle (Read completo): al dar clic en “Detalle” debe mostrarse toda la información del vehículo.
  • Borrar (Delete): al dar clic en “Borrar” debe eliminar el vehículo y mostrarse un mensaje de confirmación.
Importante: En esta web se describe el ejercicio (front). La ejecución real se hace en tu proyecto Flask, controlando operaciones con el parámetro d (base64).

2. Objetivos específicos de la Parte II

En esta Parte II el estudiante debe lograr:

  • Mostrar el listado de vehículos como pantalla principal.
  • Hacer funcionar los botones de acciones:
    • Detalle: debe abrir una vista con la información completa del vehículo.
    • Borrar: debe eliminar el registro y mostrar confirmación o error.
  • Agregar un navbar que incluya acceso al módulo de Matrículas desde cualquier pantalla.

3. Navbar obligatorio con acceso a Matrículas

Se debe agregar un navbar superior (menú) que contenga como mínimo dos accesos:

  • Vehículos: regresa al listado principal.
  • Matrículas: abre el módulo de matrículas (lista de matrículas).
Regla del ejercicio: el acceso a Matrículas debe usar el mismo mecanismo de operación por parámetro d (codificado en base64) que el módulo de Vehículos.

4. Parámetro d (base64): operación / id

El sistema controla todas las acciones usando un parámetro llamado d. Este parámetro representa: operación y id con el formato:

Formato: operacion/id (codificado en base64)

Operaciones principales usadas en esta Parte II:

  • det/id: mostrar el detalle del vehículo.
  • del/id: borrar el vehículo.
  • mat/0 (o equivalente): abrir el módulo de matrículas desde el navbar.

5. READ: Listado + Detalle funcionando

La pantalla principal debe ser una tabla con vehículos, mostrando la información básica y un bloque de Acciones.

Al presionar el botón Detalle, el sistema debe:

  • Recibir la operación det/id.
  • Consultar el vehículo en la base de datos.
  • Mostrar la vista detalle con: placa, marca, motor, chasis, combustible, año, color, avalúo, foto.
  • Mostrar también el valor de matrícula calculado a partir del avalúo.
  • Incluir un botón Regresar al listado.
✅ Evidencia: al dar clic en “Detalle”, sí debe cambiar la pantalla y mostrarse la información completa.

6. DELETE: Botón Borrar funcionando

Al presionar el botón Borrar, el sistema debe:

  • Recibir la operación del/id.
  • Eliminar el vehículo.
  • Mostrar un mensaje de confirmación: “El registro se eliminó correctamente” o un error si no se pudo.
  • Incluir botón Regresar al listado.

7. Evidencia (video o capturas)

Para validar la Parte II se debe evidenciar:

  • Listado con botones Detalle y Borrar visibles.
  • Click en Detalle → se muestra la vista detalle.
  • Click en Borrar → se elimina y aparece confirmación.
  • Navbar visible con acceso a Matrículas.
