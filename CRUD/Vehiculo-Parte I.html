Módulo CRUD - Paso 01

Vehiculo - Parte I

Configuración de base de datos y modelos de datos.

Volver a CRUD

1. Enunciado del Ejercicio (Vehículo - Parte I)

En este módulo construiremos un sistema de matriculación de vehículos aplicando el enfoque CRUD (Create, Read, Update, Delete) usando Python con Flask como backend y MySQL como base de datos.

Importante: Esta es la Parte I del proyecto, por lo tanto NO se implementa todavía el CRUD completo. En esta etapa el objetivo es preparar el módulo “Vehículos” con la base necesaria para continuar en las siguientes partes.

En esta Parte I el estudiante aprenderá a:

  • Preparar la base de datos ejecutando el script SQL del proyecto y comprendiendo las relaciones del modelo entidad–relación.
  • Mostrar el listado de vehículos como pantalla inicial del módulo (vista principal).
  • Manejar rutas controladas por parámetro utilizando una ruta por GET llamada d, que representa una operación y un id con el formato: C/1, R/5, U/3, D/2 (operación/id).
Ruta por parámetro: el sistema lee el parámetro d y lo divide en: operación y id. Por ejemplo: d=C/1.

Cuando la operación es C (Create), se despliega el formulario de registro de vehículo. Este formulario permite ingresar y enviar la información por POST al servidor:

  • Placa
  • Marca (combo cargado desde la base de datos)
  • Motor
  • Chasis
  • Combustible (selección tipo radio: por ejemplo Gasolina / Diésel / Eléctrico, según el catálogo)
  • Año (combo dinámico generado automáticamente)
  • Color (combo cargado desde la base de datos)
  • Foto (upload / carga de archivo)
  • Avalúo

En esta Parte I el formulario se deja preparado y funcional a nivel de interfaz y rutas, para que en la Parte II se complete el guardado real en la base de datos y en la Parte III se cierre el CRUD.

Alcance Parte I: Base de datos + listado + formulario (sin guardar / sin editar / sin eliminar).
Parte II: Conexión Flask ↔ MySQL + guardar (Create).
Parte III: CRUD completo (Read / Update / Delete) + mejoras.

2. Modelo de Base de Datos (ER)

A continuación se presenta el modelo entidad–relación del sistema. Este modelo define cómo se organiza la información y será la base para las operaciones CRUD que se implementarán posteriormente en Flask.

  • marca y color: catálogos para clasificar vehículos.
  • vehiculo: tabla principal (placa, motor, chasis, combustible, año, avalúo, etc.).
  • agencia: puntos de atención donde se realiza la matriculación.
  • matricula: registra la matrícula de un vehículo en un año y en una agencia.
