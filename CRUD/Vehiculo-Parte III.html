Vehículo - Parte III

Finalizando el ciclo de vida del dato: Actualización (Update) y Eliminación (Delete) con manejo de integridad referencial.

1. Lógica de Actualización

Para editar, reutilizamos el formulario de creación. La diferencia fundamental es que capturamos el ID del vehículo (que viene en un campo oculto) para saber qué registro modificar en la base de datos.

Nota Clave: La sentencia SQL debe incluir obligatoriamente el WHERE id = ..., de lo contrario, se actualizarían todos los registros de la tabla.
vehiculo.py (Método Update) Python 
def update_vehiculo(self, POST):
    # Recuperamos el ID oculto del formulario
    id_vehiculo = POST.get("id")
    
    sql = f"""
        UPDATE vehiculo SET
            placa = '{POST.get("placa")}',
            marca = {POST.get("marca")},
            motor = '{POST.get("motor")}',
            anio  = '{POST.get("anio")}',
            avaluo = {POST.get("avaluo")}
        WHERE id = {id_vehiculo};
    """
    self.con.query(sql)

2. Lógica de Eliminación

La eliminación es crítica. Antes de borrar el vehículo, debemos asegurarnos de mantener la Integridad Referencial de la base de datos.

Si borramos un vehículo que tiene matrículas registradas, la base de datos dará error. Por eso, el código primero elimina las "hijas" (matrículas) y luego al "padre" (vehículo).

vehiculo.py (Método Delete) Integridad Referencial 
def delete_vehiculo(self, id):
    # 1. Eliminación en cascada manual
    # Primero borramos las matrículas asociadas
    self.con.query(f"DELETE FROM matricula WHERE vehiculo={id};")      
    
    # 2. Ahora sí podemos borrar el vehículo
    self.con.query(f"DELETE FROM vehiculo WHERE id={id};")
    
    return "Vehículo eliminado correctamente"
