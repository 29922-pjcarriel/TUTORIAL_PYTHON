Funciones en Python

Una función es un bloque de código reutilizable que realiza una tarea específica. En Python, se definen con la palabra clave def.

Sintaxis Básica
def nombre_funcion(parametro1, parametro2):
    # Bloque de código
    resultado = parametro1 + parametro2
    return resultado
Tipos de Funciones
Sin retorno (Void)

Realizan una acción pero no devuelven un valor.

def saludar(nombre):
  print(f"Hola {nombre}")
Con retorno (Return)

Devuelven un dato para ser usado después.

def sumar(a, b):
  return a + b
Parámetros y Argumentos

Python es muy flexible con los parámetros:

  • Posicionales: Se asignan en orden. restar(10, 5)
  • Nombrados (Keyword): Se asignan por nombre. restar(b=5, a=10)
  • Por defecto: Valores opcionales. def saludar(msg="Hola"):

Manejo de Formularios con Flask

El manejo de formularios constituye uno de los mecanismos fundamentales de interacción entre el usuario y una aplicación web. A través de ellos es posible recolectar datos, validarlos y enviarlos al servidor para su procesamiento.

En aplicaciones desarrolladas con Python, esta funcionalidad se implementa comúnmente mediante el microframework Flask, el cual permite recibir solicitudes HTTP provenientes de formularios HTML y manipular sus datos en el backend.

El proceso general involucra tres etapas principales:

  • Captura de datos: El usuario introduce información en el formulario HTML.
  • Envío al servidor: Los datos se transmiten mediante métodos HTTP como GET o POST.
  • Procesamiento: Flask recibe la información usando el objeto request y ejecuta la lógica correspondiente.

Flask ofrece distintas formas de acceder a la información dependiendo del tipo de envío:

  • request.form → datos enviados mediante POST
  • request.args → datos enviados mediante GET
  • request.files → archivos cargados desde el cliente

Esta arquitectura permite construir aplicaciones web dinámicas, validando entradas del usuario, almacenando información en bases de datos o generando respuestas personalizadas.

Estructura de un formulario HTML
  • name: Identifica el campo en el backend.
  • action: Endpoint que procesará los datos.
  • method: Método HTTP (GET O POST)usado para enviar información.
                <form name="vehiculo" method="POST" action="procesa.py">
                </form>

Procesamiento con Flask

El procesamiento de formularios en Flask consiste en recibir y manipular los datos enviados por el usuario desde una interfaz HTML hacia el servidor. Este proceso forma parte del modelo cliente-servidor donde el navegador actúa como cliente y Flask como motor lógico encargado de interpretar la petición.

Cuando un formulario es enviado, el navegador genera una solicitud HTTP que puede ser de tipo GET o POST. Flask captura dicha solicitud mediante rutas definidas en la aplicación utilizando el decorador @app.route().

Dentro de la función asociada a la ruta, el objeto global request permite acceder a la información enviada:

  • request.form → Obtiene datos enviados por método POST
  • request.args → Obtiene datos enviados por método GET
  • request.files → Permite recibir archivos cargados

Este mecanismo permite implementar validaciones, almacenamiento en bases de datos, transformación de información y generación de respuestas dinámicas. Gracias a ello, Flask se convierte en una herramienta eficiente para construir aplicaciones web interactivas.

Código Flask 
from flask import Flask, render_template, request

app = Flask(__name__)

@app.route('/', methods=['GET','POST'])
def index():
    if request.method == 'POST':
        nombre = request.form['nombre']
        edad = request.form['edad']
        return f"Recibido: {nombre} - {edad}"
    return render_template('form.html')

if __name__ == '__main__':
    app.run(debug=True)

Métodos GET y POST

GET

Envía los datos a través de la URL y es utilizado en consultas simples. En Flask se accede mediante request.args.

POST

Envía los datos dentro del cuerpo de la petición HTTP, siendo más seguro para información sensible. Se accede con request.form.

Subida de Archivos

La subida de archivos permite enviar documentos, imágenes u otros recursos desde el navegador hacia el servidor. En aplicaciones desarrolladas con Flask, esto se realiza mediante formularios HTML que incluyen una configuración especial para transportar archivos binarios.

Para que el proceso funcione correctamente deben cumplirse ciertas condiciones:

  • enctype="multipart/form-data" — Permite enviar archivos en la petición HTTP.
  • request.files — Flask utiliza este objeto para acceder al archivo recibido.
  • secure_filename() — Evita problemas de seguridad con nombres maliciosos.
  • Validación — Siempre verificar tamaño, extensión y tipo del archivo.

El siguiente código pertenece al servidor Flask (app.py) y se encarga de recibir el archivo enviado desde el formulario HTML mediante request.files. Aquí se valida el nombre y se guarda de forma segura.

Backend Flask — Recepción del archivo (app.py) 
from flask import Flask, request
from werkzeug.utils import secure_filename
import os

app = Flask(__name__)

UPLOAD_FOLDER = "uploads"
app.config["UPLOAD_FOLDER"] = UPLOAD_FOLDER

@app.route("/upload", methods=["POST"])
def upload():

    # Flask recibe el archivo enviado desde HTML
    archivo = request.files['archivo']

    if archivo.filename != "":
        nombre_seguro = secure_filename(archivo.filename)

        archivo.save(
            os.path.join(
                app.config["UPLOAD_FOLDER"],
                nombre_seguro
            )
        )

    return "Archivo recibido correctamente"
