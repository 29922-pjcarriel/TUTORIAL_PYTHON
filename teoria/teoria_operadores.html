1. Operadores Aritméticos
Teoría

En Python existen los operadores aritméticos clásicos: +, -, *, /, % y la negación -a. Además, Python incluye // (división entera) y ** (potencia).

El operador / siempre devuelve un número decimal (float), aunque a y b sean enteros. Para obtener solo la parte entera del cociente se usa //.

En Flask, los campos A y B del formulario se reciben con request.form.get() y luego se aplican los operadores sobre los valores ya convertidos a número.

Operador Nombre Ejemplo Resultado
+ Suma 10 + 3 13
- Resta 10 - 3 7
* Multiplicación 10 * 3 30
/ División (Float) 10 / 3 3.333...
// División Entera 10 // 3 3
% Módulo (Residuo) 10 % 3 1
** Potencia 10 ** 3 1000
neg Negación -a -10
aritmeticos.py 
a = 10
b = 3

print(f"Suma:             a + b  = {a + b}")
print(f"Resta:            a - b  = {a - b}")
print(f"Multiplicación:   a * b  = {a * b}")
print(f"División float:   a / b  = {a / b}")     # siempre float
print(f"División entera:  a // b = {a // b}")    # solo parte entera
print(f"Módulo/Residuo:   a % b  = {a % b}")
print(f"Potencia:         a ** b = {a ** b}")
print(f"Negación:         -a     = {-a}")
▶ Salida:
Aplicación en Flask

Se obtienen los valores con a_raw = (request.form.get('a') or '').strip() y b_raw = (request.form.get('b') or '').strip(), se convierten a número y se aplica cada operador mostrando los resultados en una tabla dinámica.

2. Operadores de Asignación
Teoría

Los operadores de asignación se usan para actualizar variables combinando una operación con la asignación: +=, -=, *=, /=, %=, etc.

A diferencia de PHP, en Python no existe el operador .= para concatenar strings. En Python se usa += con el operador + para ese propósito.

Cada operador de asignación parte siempre del valor original de A. Por ejemplo, a += b equivale a a = a + b.

Operador Expresión Equivale a Resultado (A=10, B=5)
= a = A a = 10 10
+= a += B a = a + 5 15
-= a -= B a = a - 5 5
*= a *= B a = a * 5 50
/= a /= B a = a / 5 2.0
%= a %= B a = a % 5 0
+=1 a += 1 simula ++a 11
-=1 a -= 1 simula --a 9
asignacion.py 
a = 10   # valor original
b = 5
print(f"Asignación simple: a = {a}")

a0 = a; a0 += b
print(f"Suma-asignación:   a += b  → {a0}")

a0 = a; a0 -= b
print(f"Resta-asignación:  a -= b  → {a0}")

a0 = a; a0 *= b
print(f"Mult.-asignación:  a *= b  → {a0}")

a0 = a; a0 /= b
print(f"Div.-asignación:   a /= b  → {a0}")

a0 = a; a0 %= b
print(f"Mód.-asignación:   a %= b  → {a0}")

a0 = a; a0 += 1
print(f"Simula ++a:        a += 1  → {a0}")

a0 = a; a0 -= 1
print(f"Simula --a:        a -= 1  → {a0}")
▶ Salida:
En el código Flask

asign.append({"op": "+=", "expr": "a += B", "res": _fmt(a0 + b)}) — Se construye dinámicamente la tabla de resultados partiendo siempre del valor original de a.

3. Operadores de Comparación
Teoría

Python utiliza ==, !=, <, >, <=, >= para comparar valores. Siempre devuelven un booleano (True o False).

A diferencia de PHP, en Python no existe === (igualdad estricta de tipo). Python distingue entre igualdad (==) e identidad (is): el operador is verifica si dos variables apuntan al mismo objeto en memoria.

En Flask, se usa if a is None para validar si la conversión de texto a número falló, y así mostrar mensajes de error cuando los campos están vacíos.

Operador Significado Ejemplo (A=10, B=20) Resultado
== Igualdad A == B False
!= Diferencia A != B True
> Mayor que A > B False
< Menor que A < B True
>= Mayor o igual A >= B False
<= Menor o igual A <= B True
is Identidad (mismo objeto en memoria) a is None False
comparacion.py 
a = 10
b = 20

print(f"a == b  (igualdad)       → {a == b}")
print(f"a != b  (diferencia)     → {a != b}")
print(f"a > b   (mayor que)      → {a > b}")
print(f"a < b   (menor que)      → {a < b}")
print(f"a >= 10 (mayor o igual)  → {a >= 10}")
print(f"a <= 10 (menor o igual)  → {a <= 10}")

# Identidad vs igualdad (uso típico en Flask)
x = None
print(f"\nx is None  (identidad)   → {x is None}")
print(f"x == None  (igualdad)    → {x == None}")

# Validación como en Flask
def validar_campo(valor, nombre):
    if valor is None:
        return f"Error: {nombre} debe ser un número"
    return f"{nombre} válido: {valor}"

print("\n" + validar_campo(None, "A"))
print(validar_campo(10, "A"))
▶ Salida:
PHP vs Python

En PHP existe === que compara valor Y tipo al mismo tiempo. En Python no existe ese operador: se usa == para igualdad de valor e is solo para verificar identidad de objetos (especialmente útil con None).

4. Operadores Lógicos
Teoría

Los operadores lógicos en Python son and, or y not. Equivalen a &&, || y ! de PHP.

XOR (exclusivo): Python no tiene un operador XOR lógico nativo, pero cuando se trabaja con booleanos se puede usar ^ o comparar desigualdad (!=) entre booleanos para el mismo efecto.

En Flask, se combinan con el estado del Checkbox "Licencia" obtenido con licencia = bool(request.form.get('licencia')).

  • and
    AND (Y lógico) — equivale a && en PHP Devuelve True solo si ambas condiciones son verdaderas. Si la primera es False, Python no evalúa la segunda (cortocircuito).
  • or
    OR (O lógico) — equivale a || en PHP Devuelve True si al menos una condición es verdadera. Si la primera es True, no evalúa la segunda (cortocircuito).
  • not
    NOT (negación) — equivale a ! en PHP Invierte el valor booleano: si era True pasa a False y viceversa.
  • xor
    XOR (exclusivo) — usando ^ o != Devuelve True solo si exactamente uno de los dos valores es verdadero. Se simula con bool1 ^ bool2 o bool1 != bool2.
logicos.py 
a = 10
b = 5
licencia = True   # simula checkbox "Tiene licencia"

c1 = a > b       # True (10 > 5)
c2 = licencia    # True

print(f"a > b                    → {c1}")
print(f"licencia                 → {c2}")
print(f"(a>b) and licencia       → {c1 and c2}")
print(f"(a>b) or  licencia       → {c1 or c2}")
print(f"not (a > b)              → {not c1}")
print(f"(a>b) XOR licencia (^)   → {c1 ^ c2}")

# Cortocircuito en Python
print("\n--- Cortocircuito ---")
print(f"False and True  → {False and True}")   # no evalúa la derecha
print(f"True  or  False → {True or False}")     # no evalúa la derecha
▶ Salida:
En el código Flask

c2 = licencia obtenido de bool(request.form.get('licencia')). Luego: log.append({"op": "and", "expr": "(A > B) and Licencia", "res": (c1 and c2)}).

5. Incremento y Decremento
Teoría

En Python no existen los operadores ++ ni -- que son comunes en PHP, JavaScript y C. Se reemplazan por += 1 y -= 1 respectivamente.

En la interfaz Flask esto se muestra como una simulación dentro de la tabla de asignación, dejando claro que Python usa una sintaxis más explícita y legible.

¡Python NO tiene ++ ni --!

Si escribes a++ en Python obtendrás un SyntaxError. Usa siempre a += 1 para incrementar y a -= 1 para decrementar.

incremento_decremento.py 
a = 10
print(f"Valor inicial: {a}")

# Simulación de ++a
a += 1
print(f"a += 1  (simula ++a): {a}")

# Simulación de --a
a -= 1
print(f"a -= 1  (simula --a): {a}")

# Uso en bucle (como for con ++ en C/PHP)
print("\nContador con += 1:")
contador = 0
for i in range(5):
    contador += 1
    print(f"  Vuelta {i+1}: contador = {contador}")
▶ Salida:
En el código Flask

asign.append({"op": "+=1", "expr": "a += 1 (simula ++a)", "res": _fmt(a0 + 1)})

6. Operadores de Cadena (Concatenación)
Teoría

Python usa + para concatenar strings o f-strings (recomendado por ser más legible y eficiente). En Jinja2 (motor de plantillas de Flask) se usa ~ para concatenar variables en las plantillas HTML.

Las f-strings se escriben con una f antes de las comillas y permiten insertar expresiones Python directamente dentro de {}: f"Hola {nombre}".

Método Expresión Resultado
+ "Hola " + texto "Hola Mundo"
f"" f"Resultado de {A} + {B} es {A+B}" "Resultado de 10 + 5 es 15"
join ", ".join(lista) Concatenación limpia de lista
~ var1 ~ var2 (Jinja2) Solo en plantillas Flask
cadenas_concatenacion.py 
a = 10
b = 5
texto = "Mundo"

# Concatenación simple con +
saludo = "Hola " + texto
print(f"Concatenación (+):   {saludo}")

# F-string (recomendado en Python)
msg = f"El resultado de {a} + {b} es {a + b}"
print(f"F-string:            {msg}")

# Concatenación con +=
frase = "Python"
frase += " es genial"
print(f"Usando +=:           {frase}")

# join: concatenación limpia de lista (patrón Flask)
accesorios = ["Radio", "Aros", "Luces LED"]
print(f"join (lista):        {', '.join(accesorios)}")

# Repetición con *
print(f"Repetición (*):      {'—' * 25}")
▶ Salida:
En el código Flask

Se usan f-strings para mostrar resultados: f"El resultado de {_fmt(a)} + {_fmt(b)} es {_fmt(a+b)}". Para listas de checkboxes se usa "<br>".join(accesorios).

7. Operador Ternario (Condicional en una línea)
Teoría

El operador ternario de Python permite escribir una condición completa en una sola línea. Su formato es:

valor_si_verdadero   if   condicion   else   valor_si_falso

En Flask se evalúa el campo "Edad" con: res = ("Mayor de edad" if edad >= 18 else "Menor de edad"). El valor entero lo procesa _to_int(edad_raw).

Operador Expresión Edad = 17 Edad = 25
?: "Mayor" if edad >= 18 else "Menor" "Menor de edad" "Mayor de edad"
ternario.py 
# Ternario básico — evaluación de edad (como en Flask)
edad = 17
res = "Mayor de edad" if edad >= 18 else "Menor de edad"
print(f"Edad {edad}: {res}")

edad = 25
res = "Mayor de edad" if edad >= 18 else "Menor de edad"
print(f"Edad {edad}: {res}")

# Ternario con None (validación Flask)
edad_raw = None
res2 = "Edad no ingresada" if edad_raw is None else f"Edad: {edad_raw}"
print(f"Campo vacío:  {res2}")

# Ternario encadenado
nota = 75
cal = "Excelente" if nota >= 90 else "Aprobado" if nota >= 60 else "Reprobado"
print(f"Nota {nota}: {cal}")

# Ternario con lista (patrón Flask)
accesorios = ["Radio", "Aros"]
tiene = "Sí" if accesorios else "No"
print(f"¿Tiene accesorios?: {tiene}")
▶ Salida:
Tip

El ternario de Python se lee de forma natural: "dame este valor si se cumple la condición, de lo contrario dame este otro". Es más legible que el ? : de PHP.

8. Operadores de Pertenencia
Teoría

Permiten verificar si un elemento está dentro de una secuencia (lista, tupla, string).

Operador Descripción Ejemplo
in Devuelve True si el valor está presente. "a" in "hola" (True)
not in Devuelve True si el valor NO está. 5 not in [1,2,3] (True)
9. Ejemplos Típicos dentro de Flask
Contexto

En Flask, muchas comparaciones y validaciones se hacen con request.form y request.files. Los operadores vistos en los temas anteriores se aplican directamente sobre los datos que llegan del formulario HTML.

Dos patrones muy frecuentes son: la concatenación limpia con join para construir listas de selecciones de checkboxes, y el uso de is not None para validar archivos subidos.

flask_accesorios.py — Checkboxes + join + ternario 
# from flask import request  ← en Flask real

# Simulamos request.form.get()
class MockForm:
    def get(self, key):
        data = {"radio": "on", "aros": "on"}   # checkboxes marcados
        return data.get(key)

form = MockForm()

# Operador lógico: if con resultado de form.get
accesorios = []
if form.get("radio"):
    accesorios.append("Radio")
if form.get("aros"):
    accesorios.append("Aros")

# join: concatenación limpia de lista
accesorios_html = ", ".join(accesorios)

# Ternario: valor condicional en una línea
tiene_accesorios = "Sí" if accesorios else "No"

print(f"Accesorios seleccionados: {accesorios_html}")
print(f"¿Tiene accesorios?:       {tiene_accesorios}")
▶ Salida:
flask_archivos.py — Validación con is not None 
# from flask import request  ← en Flask real

# Simulamos request.files.get()
class MockFile:
    def __init__(self, name):
        self.filename = name

def procesar_foto(foto):
    # Operadores: is not None  +  and  +  comparación de string
    if foto is not None and foto.filename:
        print(f"  ✓ Archivo válido recibido: {foto.filename}")
        # aquí se procesaría el archivo...
    else:
        print("  ✗ No se subió ningún archivo o el nombre está vacío")

print("Caso 1 — foto = None:")
procesar_foto(None)

print("Caso 2 — foto con nombre vacío:")
procesar_foto(MockFile(""))

print("Caso 3 — foto válida:")
procesar_foto(MockFile("perfil.jpg"))
▶ Salida:
Resumen de patrones Flask

Los operadores de Python se combinan constantemente en Flask: is not None (identidad + negación), and/or para validaciones compuestas, join para concatenar listas de checkboxes, y el ternario para asignar valores condicionales en una sola línea.
