Sintaxis y Comentarios

La regla de oro de Python: la indentación no es estética, es obligatoria. Aprende a escribir código claro y bien documentado usando comentarios correctos en Python y HTML.

La indentación en Python

En Python, la indentación (sangría) no es “decoración”: es parte de la sintaxis. Sirve para indicar qué líneas pertenecen al mismo bloque (por ejemplo, el contenido de un if, un for o una función). Si falta sangría o la sangría es inconsistente, Python no puede interpretar correctamente el bloque.

Correcto
if 5 > 2:
    print("Cinco es mayor")
El bloque dentro del if está indentado.
Error (IndentationError)
if 5 > 2:
print("Cinco es mayor")
Falta sangría: Python no sabe qué pertenece al bloque.
Recomendación: usa 4 espacios por nivel de indentación. Evita mezclar tabulaciones con espacios.

¿Qué crea un bloque en Python?

Un bloque es un conjunto de instrucciones que se ejecutan juntas. En Python, normalmente un bloque comienza cuando una línea termina con : y las siguientes líneas quedan indentadas. Cuando vuelves “sin sangría”, significa que el bloque terminó.

Ejemplos típicos que crean bloques: if, for, while, def, class, try.

Ejemplo: condicional
edad = 18

if edad >= 18:
    print("Puedes votar")
else:
    print("Aún no")
Ejemplo: bucle
for i in range(3):
    print("Vuelta:", i)

print("Fin del bucle")

Errores comunes (y cómo evitarlos)

if 10 > 5
    print("ok")
En Python, los : significan: “aquí empieza un bloque”. Debe ser if 10 > 5:. Sin : Python no inicia el bloque.

Aunque “se vea igual”, una tabulación y 4 espacios no son lo mismo. Si mezclas ambos, puede aparecer: TabError: inconsistent use of tabs and spaces.

  • Configura tu editor para insertar 4 espacios al presionar TAB.
  • En VS Code: “Convert Indentation to Spaces”.

  print("Hola")

Solo se indenta después de abrir un bloque con :. Si indentas una línea sin haber abierto un bloque antes, puede salir: IndentationError: unexpected indent.

Comentarios en Python

Los comentarios son ignorados por el intérprete, pero son clave para que tu código sea entendible. Sirven para explicar qué hace el código y, sobre todo, por qué se tomó una decisión. Hay dos formas comunes: comentario de una línea y docstring.

1) Comentario de una línea
# Se usa strip() para evitar espacios al inicio/fin
nombre = nombre.strip()
print("Hola", nombre)
Ideal para explicar decisiones rápidas o validaciones.
2) Docstring (multilínea)
"""
Calcula el total con IVA.

Args:
  subtotal (float): valor sin IVA.
Returns:
  float: total final.
"""
Se usa en funciones, clases y módulos (documentación formal).
Tip práctico: Un comentario bueno explica el “por qué”, no lo obvio. Mejor: “Se valida para evitar datos vacíos” que “Se valida un if”.

Comentarios en HTML

En HTML los comentarios se escriben así y no se muestran en la página, pero sí quedan en el código fuente:

<!-- Este es un comentario en HTML -->
<h1>Contenido visible</h1>
¿Para qué sirve?
  • Marcar secciones del documento
  • Explicar estructura o componentes
  • Dejar notas para tu equipo
Cuidado
  • No pongas contraseñas o datos sensibles
  • El navegador no lo muestra, pero cualquiera puede verlo en “Ver código fuente”

¿Cómo construimos HTML dinámico con F-Strings?

En Flask (y en general en Python), a veces necesitamos generar HTML que cambie según datos reales (por ejemplo: un id, una operación op, datos de una base de datos, etc.). A eso se le llama HTML dinámico.

La idea es simple: armamos el HTML como texto en una variable, y con f-strings insertamos valores usando { }.

1) Base: el HTML se guarda en una variable

Paso 1: iniciar
html = ""
html += "<div>Hola</div>"
Aquí el HTML es fijo (no depende de variables).
Paso 2: hacerlo dinámico (f-string)
nombre = "Josue"
html += f"<div>Hola {nombre}</div>"
{nombre} se reemplaza por el valor real.

2) Ejemplo real: inputs hidden (como en tu proyecto)

Este patrón se usa muchísimo en formularios: guardar valores en inputs ocultos para luego procesarlos.

Correcto (con f-string)
id = 7
op = "editar"

html = ""
html += f"""
<input type="hidden" name="id" value="{id}">
<input type="hidden" name="op" value="{op}">
"""
Insertas valores dinámicos con {id} y {op}.
Incorrecto (sin la f)
id = 7
html = ""
html += """
<input type="hidden" name="id" value="{id}">
"""
Imprime {id} literal porque falta la f.

3) Insertar HTML generado por una función (combo dinámico)

En proyectos con clases (POO), es común que una función genere un <select> completo. Con f-strings puedes insertar ese HTML directamente dentro del layout.

html += f"""
<div class="col-sm-4 form-group">
  <label for="marca">Marca</label>
  {self.__get_combo_db("marca","id","descripcion","marca", self.__marca)}
</div>
"""
Todo lo que va dentro de { ... } se evalúa en Python, y el resultado se inserta como HTML.

Errores comunes (HTML dinámico)

Si no pones la f, Python no reemplaza nada y deja las llaves tal cual.

html += "<p>Hola {nombre}</p>" # queda literal

html += f"<p>{nombre</p>"

Falta cerrar }. Esto genera error de sintaxis.

Para HTML multilínea, usa f""" ... """ y así evitas problemas con comillas.

html += f"""
<div class="card">Hola {nombre}</div>
"""
Ojo: No insertes datos del usuario sin validar. Si el valor viene de un formulario, límpialo para evitar contenido raro en el HTML.
