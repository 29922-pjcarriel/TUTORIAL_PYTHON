¿Qué es Python?

Python es un lenguaje de programación de alto nivel, interpretado y de propósito general, diseñado con un enfoque claro en la legibilidad y la simplicidad. Su sintaxis cercana al lenguaje humano permite escribir programas claros, ordenados y fáciles de mantener.

A diferencia de otros lenguajes más complejos, Python prioriza que el desarrollador se concentre en resolver problemas y no en la sintaxis. Esta filosofía lo ha convertido en uno de los lenguajes más utilizados tanto en la academia como en la industria.

Características principales de Python
Sintaxis clara y legible

Python utiliza una sintaxis sencilla y cercana al lenguaje humano, lo que facilita la lectura del código y reduce errores, especialmente en proyectos grandes.

Lenguaje interpretado

El código se ejecuta directamente sin necesidad de compilación, lo que permite desarrollar, probar y depurar aplicaciones de forma más rápida.

Multiparadigma

Python soporta programación procedural, orientada a objetos y funcional, adaptándose a diferentes estilos y necesidades.

Amplio ecosistema

Cuenta con miles de librerías y frameworks que permiten desarrollar aplicaciones web, sistemas de datos, inteligencia artificial y automatización.

Uso académico y profesional

Python es ampliamente utilizado en universidades y empresas tecnológicas, lo que garantiza una gran comunidad y abundantes recursos de aprendizaje.

Alta productividad

Permite desarrollar soluciones completas con menos líneas de código, acelerando el tiempo de desarrollo sin sacrificar calidad.

Historia de Python

1989
El origen de Python

Guido van Rossum inicia el desarrollo de Python durante las vacaciones de Navidad en los Países Bajos, con el objetivo de crear un lenguaje simple y legible.

1991
Primera versión pública

Python se publica oficialmente, incorporando módulos, funciones y manejo de excepciones.

2000
Python 2.0

Se introducen listas por comprensión, recolección de basura y mejoras significativas de rendimiento.

2008
Python 3.0

Rediseño profundo del lenguaje, priorizando Unicode y claridad sintáctica, aunque rompe compatibilidad con Python 2.

2020
Fin de Python 2

Python 2 deja de recibir soporte oficial, consolidando Python 3 como estándar global.

Actualidad
Python en la actualidad

Python es uno de los lenguajes más utilizados del mundo, dominante en desarrollo web, ciencia de datos, inteligencia artificial y automatización.

Idea clave: La historia de Python demuestra que un lenguaje bien diseñado puede evolucionar sin perder simplicidad.

Arquitectura Web de Python

La arquitectura web de Python define la forma en que una aplicación escrita en Python recibe, procesa y responde a solicitudes provenientes de un navegador web.

Esta arquitectura permite separar responsabilidades, mejorar el rendimiento y facilitar el mantenimiento de aplicaciones web modernas.

Cliente

Navegador o aplicación que envía solicitudes HTTP al servidor web.

Servidor

Recibe la solicitud y la envía a la aplicación Python mediante un estándar de comunicación.

Aplicación Python

Procesa la lógica del sistema y genera la respuesta para el cliente.

¿Qué es WSGI?

WSGI (Web Server Gateway Interface) es un estándar que define cómo un servidor web se comunica con una aplicación escrita en Python.

En lugar de que cada framework tenga su propia forma de comunicarse con el servidor, WSGI establece una interfaz común y reutilizable.

Idea clave: WSGI actúa como un puente entre el servidor web (como Nginx o Apache) y la aplicación Python.
WSGI y Flask

Flask es un framework web que implementa el estándar WSGI. Esto significa que Flask puede ejecutarse sobre distintos servidores web sin modificar el código. 

from flask import Flask

app = Flask(__name__)

@app.route("/")
def home():
    return "Hola desde Flask"

En este ejemplo, Flask actúa como la aplicación WSGI que recibe la solicitud del servidor y devuelve una respuesta.

HTML y Motores de Plantillas

HTML (HyperText Markup Language) es el lenguaje encargado de definir la estructura visual de una página web: textos, enlaces, formularios, tablas y otros elementos que el usuario ve en el navegador.

En una aplicación web moderna, HTML no funciona de forma aislada. Se combina con el backend para mostrar información dinámica, personalizada y proveniente del servidor.

HTML Estático

El contenido es fijo y no cambia. Cada usuario ve exactamente la misma información, sin interacción con el backend.

HTML Dinámico

El contenido se genera en el servidor. La información cambia según el usuario, los datos o la lógica del sistema.

¿Qué es un motor de plantillas?

Un motor de plantillas es una herramienta que permite combinar HTML con datos provenientes del backend, sin mezclar directamente la lógica de programación con la vista.

Gracias a los motores de plantillas, el backend puede enviar información a HTML y generar páginas dinámicas de forma ordenada.

Idea clave: El motor de plantillas actúa como un intermediario entre Python y HTML.
Jinja2 en Flask

Jinja2 es el motor de plantillas utilizado por Flask. Permite insertar variables, estructuras de control y reutilizar componentes dentro de archivos HTML. 

from flask import Flask, render_template

app = Flask(__name__)

@app.route("/")
def home():
    return render_template("home.html", nombre="Stalin")

Ecosistema de Frameworks en Python

Un framework es una estructura de trabajo que proporciona herramientas, reglas y componentes para desarrollar aplicaciones de forma más rápida, organizada y segura.

En lugar de crear todo desde cero, los frameworks permiten enfocarse en la lógica del negocio, reutilizando soluciones ya probadas.

Python cuenta con un ecosistema amplio y diverso de frameworks que se adaptan a diferentes tipos de proyectos, desde aplicaciones web completas hasta APIs de alto rendimiento.

Python ofrece un ecosistema diverso de frameworks que se adaptan a distintos tipos de proyectos, desde aplicaciones completas hasta APIs de alto rendimiento.

Django

Framework completo y robusto. Ideal para aplicaciones grandes, con autenticación, ORM y panel administrativo integrados.

Flask

Micro-framework ligero y flexible. Perfecto para aprender backend, proyectos modulares y aplicaciones personalizadas.

FastAPI

Framework moderno enfocado en APIs. Destaca por su alto rendimiento, validación automática y documentación interactiva.

Pyramid

Framework flexible que puede iniciar pequeño y escalar a aplicaciones empresariales según las necesidades del proyecto.

Idea clave: No existe un único framework perfecto; Python permite elegir la herramienta adecuada según el tipo y complejidad del proyecto.
¿Cuál framework usar?
  • Django: aplicaciones grandes y completas
  • Flask: aprendizaje, proyectos modulares y flexibles
  • FastAPI: APIs modernas y alto rendimiento
  • Pyramid: proyectos que crecen con el tiempo
