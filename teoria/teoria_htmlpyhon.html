¿Qué significa “Python en la Web”?

Cuando hablamos de Python en la Web, nos referimos a usar Python para crear un servidor que recibe peticiones desde un navegador y responde con contenido (por ejemplo, una página HTML, un JSON o una redirección).

Idea clave: El HTML se muestra en el navegador, pero la lógica (validaciones, consultas a base de datos, seguridad, etc.) ocurre en el servidor.

Flujo real: Navegador Flask Respuesta

1) Navegador

El usuario abre una URL y envía una solicitud (HTTP).

2) Flask

Ejecuta la lógica (validaciones, cálculos, BD, seguridad).

3) Respuesta

El servidor responde: HTML, JSON o redirección.

HTTP en 5 minutos (lo esencial)

HTTP es el “idioma” que usan navegador y servidor para comunicarse. Cada petición tiene un método y una URL, y el servidor devuelve una respuesta con un código de estado.

Request (Petición)
  • Método: GET / POST
  • URL: /teoria/html-python-web
  • Headers: info extra
  • Body: datos (en POST)
Servidor (Lógica)
  • Lee datos
  • Valida reglas
  • Consulta BD (si aplica)
  • Genera respuesta
Response (Respuesta)
  • Status: 200, 404, 500…
  • HTML: página
  • JSON: APIs
  • Redirect: a otra ruta
Regla práctica: GET para ver (mostrar), POST para enviar (guardar datos).

Rutas en Flask (GET y POST)

En Flask, una ruta es una URL que apunta a una función. Esa función decide qué devolver: HTML, JSON o una redirección.

Ejemplo básico (app.py):

from flask import Flask

app = Flask(__name__)

@app.route("/")
def inicio():
    return "<h1>Hola, Python Academy</h1>"

@app.route("/saludo/<nombre>")
def saludo(nombre):
    return f"<h2>Hola, {nombre}!</h2>"
Nota: En proyectos reales, en vez de escribir HTML en la función, se usa una página HTML completa. La función solo prepara datos y devuelve la vista.

Formularios: enviar datos desde HTML

Los formularios permiten que el usuario envíe información al servidor. Normalmente se usa POST.

HTML (formulario)
<form action="/login" method="POST">
  <input name="usuario" placeholder="Usuario">
  <input name="clave" type="password" placeholder="Clave">
  <button type="submit">Ingresar</button>
</form>
Envía datos del formulario al servidor.
Python (recibir datos)
from flask import request

@app.route("/login", methods=["POST"])
def login():
  u = request.form.get("usuario")
  p = request.form.get("clave")
  return f"Usuario: {u}"
Lee datos enviados desde el navegador.

Archivos estáticos: CSS, JS e imágenes

En una web real, el estilo (CSS), scripts (JS) e imágenes se sirven como archivos estáticos. Flask (y cualquier framework) recomienda tener una carpeta dedicada para esto.

Tipo Ejemplo
CSS static/css/main.css
JavaScript static/js/app.js
Imágenes static/img/logo.png

Errores comunes y cómo solucionarlos

Eso es un problema de codificación. Guarda el archivo como UTF-8 (sin BOM) y mantén <meta charset="utf-8">.

Revisa rutas relativas (../) y que los archivos existan. Abre la consola del navegador (F12) y mira si sale 404.

Mira la terminal donde corre Flask: ahí aparece el error real. Corrige la línea indicada y vuelve a reiniciar.
