Sobre Nosotros

Transformando la teoría en aplicaciones reales

Este proyecto representa el culmen del aprendizaje en Tecnologías Web, integrando lógica de programación, diseño de interfaces y gestión de datos.

Nuestro objetivo es proporcionar una guía clara y estructurada para entender cómo Python, siendo un lenguaje de propósito general, se convierte en una herramienta poderosa para la web mediante Flask.

Plan de Estudios

Nuestra Ruta de Aprendizaje

Hemos dividido el contenido en tres pilares fundamentales para garantizar una comprensión profunda de la tecnología.

1. Fundamentos Teóricos

Antes de codificar, entendemos el "por qué". Analizamos la historia de Python, su sintaxis limpia, los tipos de datos dinámicos y los paradigmas de Programación Orientada a Objetos.

  • Tipos de Datos (Listas, Tuplas, Diccionarios)
  • Estructuras de Control (If, While, For)
  • Clases, Objetos y Herencia
Ir a Teoría

2. Lógica Algorítmica

La teoría se consolida con la práctica. Presentamos una suite de 17 ejercicios que resuelven problemas clásicos de la computación, desde series matemáticas hasta validaciones complejas.

  • Serie de Fibonacci y Factoriales
  • Validación de Cédula Ecuatoriana
  • Operaciones con Matrices y Cubos
Ver Ejercicios

3. Desarrollo Web Fullstack

El proyecto final. Un sistema CRUD (Crear, Leer, Actualizar, Borrar) completo utilizando Flask como backend y MySQL como base de datos, incluyendo autenticación.

  • Conexión a Base de Datos MySQL
  • Manejo de Sesiones (Login/Logout)
  • Subida de Archivos (Imágenes)

Preguntas Frecuentes

Respuestas a las dudas comunes sobre la tecnología utilizada en este proyecto académico.

Flask es un micro-framework que nos permite entender mejor cómo funciona el ciclo de petición-respuesta HTTP sin la "magia" oculta de Django. Es ideal para propósitos educativos y proyectos modulares.

Sí, utilizamos la librería `mysql-connector` con parámetros parametrizados para prevenir inyecciones SQL, además de gestionar las credenciales de forma segura.

Implementamos una variación del patrón MVC (Modelo-Vista-Controlador), donde las rutas de Flask actúan como controladores, las plantillas Jinja2 como vistas y las clases Python como modelos de datos.
